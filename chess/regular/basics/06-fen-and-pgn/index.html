6. FEN and PGN

Beyond move-by-move algebraic notation, two further standards let players, databases, and engines store and exchange chess positions and games.

Forsyth-Edwards Notation (FEN)

FEN is a compact way to describe a single position - the placement of every piece, whose turn it is, castling rights, en passant target square, and move counters - in one line of text. This is what chess engines and websites use to set up "puzzle of the day" positions or analyse a specific moment in a game.

rnbqkbnr/pppppppp/8/8/4P3/8/PPPP1PPP/RNBQKBNR b KQkq e3 0 1
- The starting position after 1.e4

A FEN string has six fields, separated by spaces:

  1. Piece placement, rank by rank from rank 8 to rank 1, files a to h. Uppercase letters are White pieces, lowercase are Black. Numbers represent consecutive empty squares.
  2. Active colour: w or b - whose turn it is to move.
  3. Castling availability: any combination of K, Q, k, q (kingside/queenside, White/Black), or - if neither side can castle.
  4. En passant target square: the square behind a pawn that just moved two squares, or - if none.
  5. Halfmove clock: the number of halfmoves since the last capture or pawn move (used for the 50-move rule).
  6. Fullmove number: starts at 1, increments after Black's move.

Portable Game Notation (PGN)

PGN is the standard file format for storing entire games, including metadata. A PGN file contains a "tag pair" header section followed by the move list in algebraic notation:

[Event "World Championship"]
[Site "Reykjavik"]
[Date "1972.07.23"]
[White "Fischer, Robert J."]
[Black "Spassky, Boris V."]
[Result "1-0"]

1.d4 Nf6 2.c4 g6 3.Nc3 Bg7 4.e4 d6 ... 1-0

Almost every chess database, online platform, and analysis tool can import and export PGN. Learning to read a PGN header tells you the players, event, date, and result of a game at a glance - useful when studying master games such as those covered in the Master Games section.
