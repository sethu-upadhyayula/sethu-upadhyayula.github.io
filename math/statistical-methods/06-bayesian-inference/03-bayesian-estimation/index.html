6.3 Bayesian Estimation

Bayesian point estimation is framed as a decision problem: the choice of estimator depends on the loss function used to penalise errors. This section also covers non-informative priors, hierarchical modelling, and the MCMC algorithms that make Bayesian inference computationally tractable for complex models.

Loss Functions and Bayes Estimators

Theorem 6.3.1 - Bayes Estimators Under Common Loss Functions

The Bayes estimator minimises posterior expected loss $E[L(\theta, \hat{\theta}) \mid x]$:

  • Squared error loss $L(\theta,\hat\theta) = (\theta-\hat\theta)^2$: Bayes estimator $=$ posterior mean $E[\theta\mid x]$.
  • Absolute error loss $L(\theta,\hat\theta) = |\theta-\hat\theta|$: Bayes estimator $=$ posterior median.
  • 0-1 loss $L(\theta,\hat\theta) = \mathbf{1}[\theta \ne \hat\theta]$: Bayes estimator $=$ MAP estimate $\arg\max p(\theta\mid x)$.

Jeffreys Prior

Definition 6.3.2 - Jeffreys Prior

The Jeffreys prior is a non-informative prior defined as $$p(\theta) \propto \sqrt{|I(\theta)|},$$ where $I(\theta)$ is the Fisher information. It is invariant under reparametrisation: if $\phi = g(\theta)$ then the Jeffreys prior for $\phi$ equals the Jeffreys prior for $\theta$ after the change of variables.

Examples: for $\theta \in (0,\infty)$ (scale), Jeffreys prior is $p(\theta) \propto 1/\theta$ (improper); for the Binomial $p$ parameter, Jeffreys prior is $\mathrm{Beta}(1/2, 1/2)$.

Hierarchical Models

Definition 6.3.3 - Hierarchical Bayesian Model

A hierarchical model introduces multiple levels of uncertainty. For example, with group means $\theta_i$ and a common prior: $$X_{ij} \mid \theta_i \sim p(x\mid\theta_i), \quad \theta_i \mid \tau \sim p(\theta\mid\tau), \quad \tau \sim p(\tau).$$ The hyperparameter $\tau$ governs the between-group variation and is itself estimated from the data. This partial pooling shrinks group estimates toward the grand mean, reducing variance without ignoring group-level information. Hierarchical models are the Bayesian counterpart of random effects models.

Markov Chain Monte Carlo

Definition 6.3.4 - Metropolis-Hastings Algorithm

When the posterior $p(\theta\mid x)$ lacks a closed form, MCMC constructs a Markov chain with stationary distribution $p(\theta\mid x)$. The Metropolis-Hastings algorithm:

  1. Given current state $\theta^{(t)}$, propose $\theta^* \sim q(\theta^* \mid \theta^{(t)})$.
  2. Compute acceptance ratio $r = \dfrac{p(\theta^*\mid x)\,q(\theta^{(t)}\mid\theta^*)}{p(\theta^{(t)}\mid x)\,q(\theta^*\mid\theta^{(t)})}$.
  3. Set $\theta^{(t+1)} = \theta^*$ with probability $\min(1, r)$; otherwise $\theta^{(t+1)} = \theta^{(t)}$.
Only the ratio $p(\theta^*\mid x)/p(\theta^{(t)}\mid x) \propto p(x\mid\theta^*)\,p(\theta^*) / (p(x\mid\theta^{(t)})\,p(\theta^{(t)}))$ is needed, so the normalising constant $p(x)$ cancels.

Definition 6.3.5 - Gibbs Sampler

The Gibbs sampler is a special case of Metropolis-Hastings for multi-parameter posteriors $p(\theta_1,\ldots,\theta_k\mid x)$. At each step, cycle through parameters and sample each from its full conditional $p(\theta_j \mid \theta_{-j}, x)$. This requires closed-form full conditionals (available, e.g., in conjugate hierarchical models) but always accepts, making it efficient when the conditionals are easy to sample.

Model Comparison: Bayes Factor

Definition 6.3.6 - Bayes Factor

The Bayes factor for $M_1$ vs.\ $M_0$ is $$BF_{10} = \frac{p(x \mid M_1)}{p(x \mid M_0)} = \frac{\int p(x\mid\theta_1, M_1)\,p(\theta_1\mid M_1)\,d\theta_1}{\int p(x\mid\theta_0, M_0)\,p(\theta_0\mid M_0)\,d\theta_0}.$$ $BF_{10} > 1$ favours $M_1$; by Jeffreys' scale, $BF_{10} > 100$ is "decisive" evidence. Unlike AIC/BIC, the Bayes factor integrates over parameter uncertainty and automatically penalises model complexity (Occam's razor). Computing marginal likelihoods often requires numerical integration or MCMC.
