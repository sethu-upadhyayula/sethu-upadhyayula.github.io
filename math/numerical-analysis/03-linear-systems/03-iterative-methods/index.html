3.3 Iterative Methods (Jacobi, Gauss-Seidel)

For large sparse systems - arising from discretisation of PDEs, graph problems, or network models - direct methods require storing and factorising matrices with millions of rows, which is prohibitive. Iterative methods instead produce a sequence of approximations using only matrix-vector products, exploiting sparsity to achieve $O(n)$ or $O(n \log n)$ cost per iteration.

Splitting Methods

Definition - Matrix Splitting

Write $A = D + L + U$ where $D$ is the diagonal part, $L$ the strictly lower-triangular part, and $U$ the strictly upper-triangular part. A splitting $A = M - N$ leads to the iteration $Mx^{k+1} = Nx^k + b$, or $x^{k+1} = M^{-1}Nx^k + M^{-1}b$. The iteration matrix is $T = M^{-1}N$.

Definition - Jacobi Iteration

The Jacobi method takes $M = D$, $N = -(L+U)$:

$$x^{k+1} = D^{-1}\bigl(-(L+U)x^k + b\bigr), \qquad \text{i.e.,} \quad x_i^{k+1} = \frac{1}{a_{ii}}\Bigl(b_i - \sum_{j \ne i} a_{ij} x_j^k\Bigr).$$

Each component of $x^{k+1}$ is computed from the old iterate $x^k$, so all components can be updated in parallel.

Definition - Gauss-Seidel Iteration

The Gauss-Seidel method takes $M = D + L$, $N = -U$:

$$(D+L)x^{k+1} = -Ux^k + b, \qquad \text{i.e.,} \quad x_i^{k+1} = \frac{1}{a_{ii}}\Bigl(b_i - \sum_{j < i} a_{ij}x_j^{k+1} - \sum_{j > i}a_{ij}x_j^k\Bigr).$$

Components are updated sequentially, immediately using the most recently computed values. This typically halves the number of iterations compared to Jacobi, though it cannot be parallelised as easily.

Convergence

Theorem - Spectral Radius Criterion

The iteration $x^{k+1} = Tx^k + c$ converges for any starting vector $x^0$ if and only if the spectral radius $\rho(T) < 1$. The asymptotic convergence rate is $-\log_{10}\rho(T)$ decimal digits per iteration. For diagonally dominant matrices, both Jacobi and Gauss-Seidel converge. Gauss-Seidel converges in at most half as many iterations as Jacobi for most problems.

Successive Over-Relaxation (SOR)

Definition - SOR

The SOR method introduces a relaxation parameter $\omega \in (0,2)$:

$$x_i^{k+1} = (1-\omega)x_i^k + \frac{\omega}{a_{ii}}\Bigl(b_i - \sum_{ji}a_{ij}x_j^k\Bigr).$$

For the model Laplacian problem on a $n \times n$ grid, the optimal parameter is: $$\omega^* = \frac{2}{1 + \sqrt{1 - \rho_J^2}},$$ where $\rho_J$ is the spectral radius of the Jacobi iteration matrix. With $\omega^*$, SOR converges in $O(\sqrt{\kappa})$ iterations rather than $O(\kappa)$ for Gauss-Seidel.

Conjugate Gradient Method

Theorem - Conjugate Gradient (CG)

For SPD $A$, the Conjugate Gradient method is a Krylov subspace method that minimises the $A$-norm of the error over $\mathcal{K}_k = \text{span}\{r_0, Ar_0, \ldots, A^{k-1}r_0\}$. It terminates in at most $n$ steps (exact arithmetic). The practical convergence bound is:

$$\frac{\|e_k\|_A}{\|e_0\|_A} \le 2\left(\frac{\sqrt{\kappa_2(A)}-1}{\sqrt{\kappa_2(A)}+1}\right)^k,$$

so CG reaches accuracy $\varepsilon$ in $O(\sqrt{\kappa_2(A)}\,\log(2/\varepsilon))$ iterations. With an ICCG preconditioner (incomplete Cholesky), the effective condition number is greatly reduced, often yielding mesh-independent convergence for elliptic PDE problems.

Remark - Direct vs. Iterative

Use direct methods (LU, Cholesky) for: dense matrices; moderate $n$ ($n \lesssim 10^4$); multiple right-hand sides with the same $A$; problems where exact solution (up to rounding) is required. Use iterative methods for: large sparse matrices ($n \gtrsim 10^5$); matrices where sparsity would be destroyed by factorisation; problems where a moderate-accuracy solution suffices; PDE problems with structured sparsity amenable to multigrid or FFT preconditioning.
