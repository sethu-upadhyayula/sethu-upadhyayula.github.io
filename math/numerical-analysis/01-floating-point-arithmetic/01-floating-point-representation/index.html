1.1 Floating-Point Representation

Real numbers are infinite objects; computers store only finite approximations. The IEEE 754 standard governs how virtually all modern hardware represents and computes with floating-point numbers. Understanding its structure — sign, exponent, mantissa — is the foundation for analysing numerical error.

IEEE 754 Double Precision

Definition — IEEE 754 Double Precision

A double-precision floating-point number occupies 64 bits partitioned as:

  • 1 sign bit $s \in \{0,1\}$
  • 11 exponent bits, encoding a biased exponent $e$ with bias 1023
  • 52 fraction bits $f_1 f_2 \cdots f_{52}$

For a normalised number the represented value is: $$x = (-1)^s \cdot 2^{e-1023} \cdot (1.f_1 f_2 \cdots f_{52})_2 = (-1)^s \cdot 2^{e-1023} \cdot \left(1 + \sum_{i=1}^{52} f_i \cdot 2^{-i}\right).$$ The leading 1 is implicit and not stored, giving an effective 53-bit significand (precision $p = 53$ binary digits).

Normalised Form and Range

Definition — Normalised Floating-Point Number

A floating-point number $x \ne 0$ is normalised when its significand satisfies $1 \le |m| < 2$ (base 2). In double precision, normalised numbers span the range: $$|x| \in [2^{-1022},\; (2-2^{-52})\cdot 2^{1023}] \approx [2.2\times 10^{-308},\; 1.8\times 10^{308}].$$

Machine Epsilon

Definition — Machine Epsilon

Machine epsilon $\varepsilon_{\text{mach}}$ is the smallest positive floating-point number such that $\text{fl}(1 + \varepsilon_{\text{mach}}) > 1$. For IEEE 754 double precision with round-to-nearest: $$\varepsilon_{\text{mach}} = 2^{-52} \approx 2.22 \times 10^{-16}.$$ Equivalently, for any real $x$ in the normalised range, its floating-point representation $\text{fl}(x)$ satisfies $|\text{fl}(x) - x| \le \tfrac{1}{2}\varepsilon_{\text{mach}}|x|$.

Remark — Relative Error Bound

The fundamental property of floating-point arithmetic is that each basic operation $\circ \in \{+,-,\times,\div\}$ satisfies $\text{fl}(x \circ y) = (x \circ y)(1+\delta)$ for some $|\delta| \le \varepsilon_{\text{mach}}$, provided no overflow or underflow occurs. This is the unit roundoff model.

Subnormal Numbers

Definition — Subnormal (Denormalised) Numbers

When the exponent field is all zeros, the leading bit is taken as 0 rather than 1, giving subnormal numbers: $$x = (-1)^s \cdot 2^{-1022} \cdot (0.f_1\cdots f_{52})_2.$$ Subnormals fill the gap between 0 and the smallest normalised number, enabling gradual underflow down to $2^{-1074} \approx 5\times 10^{-324}$.

Overflow, Underflow, and Special Values

Definition — Special Values

Overflow occurs when a result exceeds $(2 - 2^{-52})\cdot 2^{1023}$; the result is set to $\pm\infty$.

Underflow occurs when a positive result is smaller than $2^{-1022}$; gradual underflow produces a subnormal, and flush-to-zero underflow sets the result to 0.

IEEE 754 defines four special values via reserved bit patterns:

  • $+\infty$ and $-\infty$: exponent all-ones, fraction all-zeros.
  • NaN (Not a Number): exponent all-ones, nonzero fraction. Produced by $0/0$, $\infty - \infty$, $\sqrt{-1}$, etc. NaN propagates through arithmetic: any operation involving NaN returns NaN.
  • $\pm 0$: exponent and fraction all-zeros. $+0 = -0$ for comparison purposes, but $1/(+0) = +\infty$ while $1/(-0) = -\infty$.

Floating-Point Density

Proposition — Distribution of Floating-Point Numbers

In each binade $[2^k, 2^{k+1})$, there are exactly $2^{52}$ normalised double-precision numbers uniformly spaced with gap $2^{k-52}$. The spacing therefore doubles as the magnitude doubles: large numbers are more sparsely represented than small numbers, which is why relative error (not absolute) is the natural measure.

Example — Exact Representation

The number $0.1$ has no exact binary representation. In double precision: $$\text{fl}(0.1) = 0.1000000000000000055511151231257827021181583404541015625\ldots$$ This is an error of roughly $5.6 \times 10^{-18}$, well within $\tfrac{1}{2}\varepsilon_{\text{mach}} \times 0.1 \approx 1.1 \times 10^{-17}$.

In contrast, $0.5 = 2^{-1}$ and $0.25 = 2^{-2}$ are represented exactly, as are all integers up to $2^{53} = 9{,}007{,}199{,}254{,}740{,}992$.
