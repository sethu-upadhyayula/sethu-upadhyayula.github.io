1.2 Rounding and Cancellation Errors

Every floating-point operation introduces a small rounding error. Understanding how these errors originate, how they propagate through a computation, and when they can catastrophically amplify is essential to writing numerically reliable code.

Rounding Modes

Definition — IEEE 754 Rounding Modes

IEEE 754 specifies four rounding modes for the operation of mapping a real number $x$ to its floating-point approximation $\text{fl}(x)$:

  • Round to nearest, ties to even (default): choose the floating-point number nearest to $x$; on a tie, choose the one with even last bit.
  • Round toward $+\infty$ (round up): $\text{fl}(x) \ge x$.
  • Round toward $-\infty$ (round down): $\text{fl}(x) \le x$.
  • Round toward zero (truncation): $|\text{fl}(x)| \le |x|$.

The default mode minimises the maximum relative error and is used in almost all scientific computing.

Absolute and Relative Error

Definition — Absolute and Relative Error

Given the true value $x$ and its floating-point approximation $\tilde{x} = \text{fl}(x)$:

  • Absolute error: $|e| = |\tilde{x} - x|$.
  • Relative error: $|e_r| = |\tilde{x} - x| / |x|$ (for $x \ne 0$).

Under round-to-nearest, the fundamental rounding error bound is: $$|\text{fl}(x) - x| \le \tfrac{1}{2}\varepsilon_{\text{mach}}\,|x|,$$ so the relative error in storing any real number is at most $\frac{1}{2}\varepsilon_{\text{mach}} \approx 1.11\times 10^{-16}$ for double precision.

Error Propagation

Theorem — Unit Roundoff Model

For any basic arithmetic operation $\circ \in \{+,-,\times,\div\}$, IEEE 754 arithmetic satisfies the unit roundoff model: $$\text{fl}(x \circ y) = (x \circ y)(1 + \delta), \quad |\delta| \le \varepsilon_{\text{mach}},$$ provided no overflow or underflow occurs. Each operation introduces at most one unit of relative error.

Proposition — Error Propagation through a Chain

For a sequence of $n$ operations, the accumulated relative error is bounded by approximately $n\varepsilon_{\text{mach}}$ (using $(1+\delta_1)(1+\delta_2)\cdots(1+\delta_n) \approx 1 + \sum\delta_i$ for small $\delta_i$). More precisely, $\prod_{i=1}^n(1+\delta_i) = 1+\theta_n$ where $|\theta_n| \le \frac{n\varepsilon_{\text{mach}}}{1-n\varepsilon_{\text{mach}}}$ for $n\varepsilon_{\text{mach}} < 1$.

This is typically denoted $|\theta_n| \lesssim n\varepsilon_{\text{mach}}$ and forms the basis of rounding error analysis of algorithms.

Catastrophic Cancellation

Definition — Catastrophic Cancellation

Catastrophic cancellation occurs when two nearly equal floating-point numbers are subtracted. The leading significant digits cancel, leaving only the trailing (potentially erroneous) digits. If $a$ and $b$ agree to $k$ decimal digits, the result $a - b$ loses up to $k$ significant digits of precision.

Example — Loss of Significance

Suppose $a = 1.23456789012345$ and $b = 1.23456789012344$ (differ only in the last digit). In double precision both are stored with relative error $\sim \varepsilon_{\text{mach}}$. Their difference $a - b \approx 10^{-14}$ retains only about $10^{-16}/10^{-14} = 1\%$ correct relative precision — effectively all precision is lost.

More concretely: $\text{fl}(a) - \text{fl}(b)$ has relative error of order 1 when $|a-b|/|a| \sim \varepsilon_{\text{mach}}$.

The Quadratic Formula: A Case Study

Example — Unstable vs Stable Quadratic Formula

The roots of $ax^2 + bx + c = 0$ are nominally: $$x_{1,2} = \frac{-b \pm \sqrt{b^2 - 4ac}}{2a}.$$ When $b^2 \gg 4ac$, we have $\sqrt{b^2 - 4ac} \approx |b|$, and one of the roots involves subtracting two nearly equal quantities.

For $b > 0$, the root $x_1 = (-b + \sqrt{b^2 - 4ac})/(2a)$ suffers catastrophic cancellation. The numerically stable form uses the identity $x_1 x_2 = c/a$:

  1. Compute $x_2 = (-b - \sqrt{b^2 - 4ac})/(2a)$ (no cancellation for $b > 0$).
  2. Recover $x_1 = c/(a\,x_2)$.

Numerical example: $x^2 - (10^8 + 10^{-8})x + 1 = 0$. Roots are $x_1 \approx 10^{-8}$ and $x_2 \approx 10^8$. The naive formula gives $x_1 = (10^8 - \sqrt{10^{16} - 4})/2 \approx 0$ in double precision. The stable formula gives $x_1 = 1/x_2 = 10^{-8}$ exactly.

Kahan Summation Algorithm

Algorithm — Kahan Compensated Summation

Naive summation $S = \sum_{i=1}^n x_i$ accumulates rounding error proportional to $n\varepsilon_{\text{mach}}$. The Kahan summation algorithm reduces this to $O(\varepsilon_{\text{mach}})$ independent of $n$ by tracking a running compensation term $c$:

S = 0; c = 0
for i = 1 to n:
    y = x[i] - c          // compensated value
    t = S + y             // add to sum (low bits of y may be lost)
    c = (t - S) - y       // recover lost bits
    S = t
return S

The compensation $c$ captures the rounding error from the previous step. The total error is bounded by $2\varepsilon_{\text{mach}}\sum|x_i|$, compared to $n\varepsilon_{\text{mach}}\sum|x_i|$ for naive summation. This is crucial when summing large arrays of floating-point numbers.

Remark — Benign vs Catastrophic Cancellation

Not all cancellation is catastrophic. If both operands are exact (e.g., $\text{fl}(a) = a$ and $\text{fl}(b) = b$), then $\text{fl}(a-b) = a - b$ exactly by Sterbenz's lemma (when $a/2 \le b \le 2a$). Cancellation is only harmful when the operands themselves carry earlier rounding errors that are amplified by the subtraction.
