Chapter 4: Linear Programming

Linear programming (LP) is the special case of convex optimisation in which both the objective and constraints are linear. Its structure admits an exact combinatorial algorithm - the simplex method - that moves along vertices of the feasible polyhedron. LP is simultaneously a practically indispensable tool and the theoretical prototype for duality and interior-point theory.

Chapter Overview

4.1 - Standard Form

Any LP can be written $\min c^Tx$ subject to $Ax=b$, $x\ge0$. Inequality constraints are converted by adding slack variables. A basic feasible solution (BFS) corresponds to choosing a basis $B\subseteq\{1,\ldots,n\}$ of $m$ columns of $A$ such that $x_B=B^{-1}b\ge0$ and $x_N=0$. Vertices of the feasible polyhedron correspond exactly to BFS's.

4.2 - The Simplex Method

Optimality is checked via reduced costs $\bar c_N=c_N-c_B^TB^{-1}N$: if $\bar c_N\ge0$ the current BFS is optimal. Otherwise, select an entering variable (most negative reduced cost) and a leaving variable (minimum ratio test). Bland's rule prevents cycling. Despite exponential worst-case complexity, simplex is extremely efficient in practice.

4.3 - Revised Simplex and Big-M

The revised simplex method maintains $B^{-1}$ explicitly, reducing computational cost per pivot. When no initial BFS is available, Phase I (or the Big-M method) finds one by solving an auxiliary LP. Sensitivity analysis quantifies how the optimal value changes with perturbations to $b$ or $c$. The dual simplex method operates on dual feasibility.
