4.1 Standard Form and Basic Feasible Solutions

Linear programming (LP) studies the minimisation of a linear objective over a polyhedral feasible set. The standard form reveals the combinatorial structure of LP: optima always occur at vertices (extreme points) of the feasible polyhedron, and vertices are characterised algebraically by basic feasible solutions.

Standard Form

Definition 4.1.1 - Standard Form LP

A linear programme in standard form is $$\min_{x\in\mathbb{R}^n}\; c^\top x \quad\text{subject to}\quad Ax = b,\; x \ge 0,$$ where $A\in\mathbb{R}^{m\times n}$ (with $m \le n$), $b\in\mathbb{R}^m$, and $c\in\mathbb{R}^n$. We assume $A$ has full row rank $m$.

The feasible set is the polyhedron $\mathcal{P} = \{x \ge 0 : Ax = b\}$. The problem is feasible if $\mathcal{P}\ne\emptyset$, bounded if the optimal value is finite, and infeasible otherwise.

Conversion to Standard Form

Proposition 4.1.2 - Reduction via Slack Variables

Any LP in general form can be rewritten in standard form:

  • Inequality $Ax \le b$: introduce slack variable $s \ge 0$ so that $Ax + s = b$, $s \ge 0$.
  • Inequality $Ax \ge b$: introduce surplus variable $s \ge 0$ so that $Ax - s = b$, $s \ge 0$.
  • Free variable $x_j$: write $x_j = x_j^+ - x_j^-$ with $x_j^+, x_j^- \ge 0$.
  • Maximisation $\max c^\top x$: replace with $\min(-c)^\top x$.

These transformations preserve the optimal value (after reverting the sign for maximisation) and at most double the number of variables.

Basic Feasible Solutions

Definition 4.1.3 - Basis and Basic Feasible Solution

A basis $B \subseteq \{1,\ldots,n\}$ of size $m$ is a set of column indices such that the submatrix $A_B$ (formed by the columns in $B$) is invertible. The complementary set $N = \{1,\ldots,n\}\setminus B$ is the non-basic set.

The corresponding basic solution is obtained by setting $x_N = 0$ and solving: $$x_B = A_B^{-1}b, \quad x_N = 0.$$ It is a basic feasible solution (BFS) if additionally $x_B \ge 0$, i.e., $A_B^{-1}b \ge 0$.

A BFS is degenerate if some component of $x_B$ equals zero (a basic variable is at the boundary of its non-negativity constraint).

Theorem 4.1.4 - Vertices are Basic Feasible Solutions

A point $x\in\mathcal{P}$ is an extreme point (vertex) of the polyhedron $\mathcal{P} = \{x \ge 0 : Ax = b\}$ if and only if it is a basic feasible solution.

Proof sketch. If $x$ is a BFS with basis $B$, then $x$ cannot be written as a strict convex combination of two distinct feasible points, because any convex perturbation must keep $x_N = 0$ (forced by $x_N \ge 0$ and $x_N = 0$), and the basic components are then uniquely determined by $Ax = b$. Conversely, if $x$ is not a BFS, one can exhibit a feasible direction along which $x$ moves without leaving $\mathcal{P}$. $\square$

Theorem 4.1.5 - Existence of Optimal at a Vertex

If the LP is feasible and bounded, then there exists an optimal solution that is a basic feasible solution.

Proof sketch. The minimum of a linear function over a (non-empty, bounded) polyhedron is attained at an extreme point. Since extreme points of $\mathcal{P}$ coincide with BFS (Theorem 4.1.4), the result follows. When the polyhedron is unbounded but the problem is still bounded, the minimum is still attained because the feasible set is closed and the sublevel sets $\{x : c^\top x \le \alpha\} \cap \mathcal{P}$ are compact for $\alpha$ close to the optimum. $\square$

Degeneracy and Counting

Remark - Degeneracy

Degeneracy occurs when a BFS has fewer than $n - m$ non-zero components at their bounds, i.e., some basic variables equal zero. Geometrically, more than $n$ hyperplanes meet at the vertex. Degeneracy does not affect correctness of the simplex method but can cause cycling (infinite loops) in degenerate pivots - resolved by Bland's rule (Section 4.2).

The number of possible bases is $\binom{n}{m}$. For a non-degenerate LP, each basis determines a unique BFS, so the number of vertices is at most $\binom{n}{m}$. Typical LPs are non-degenerate after a small perturbation of $b$.

Example - BFS of a Simple LP

Consider $\min\; -x_1 - x_2$ s.t. $x_1 + x_2 + s_1 = 4$, $x_1 + s_2 = 3$, $x_1, x_2, s_1, s_2 \ge 0$ ($n = 4$, $m = 2$). Choose basis $B = \{1, 2\}$: $$A_B = \begin{pmatrix}1 & 1 \\ 1 & 0\end{pmatrix},\quad x_B = A_B^{-1}\begin{pmatrix}4\\3\end{pmatrix} = \begin{pmatrix}3\\1\end{pmatrix} \ge 0.$$ So $(x_1, x_2, s_1, s_2) = (3, 1, 0, 0)$ is a non-degenerate BFS with objective value $-4$. One can verify this is optimal by checking all other BFS.

The basis $B = \{3, 4\}$ gives $x_B = (4, 3)$, i.e., $x_1 = x_2 = 0$, which is the BFS corresponding to the origin - feasible but with objective value $0$, confirming it is not optimal.
