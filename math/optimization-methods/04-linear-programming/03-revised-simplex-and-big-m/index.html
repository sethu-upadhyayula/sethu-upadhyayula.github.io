4.3 Revised Simplex and Big-M

The revised simplex method is the numerically practical implementation of the simplex algorithm. Instead of updating the full tableau, it maintains only the basis inverse $A_B^{-1}$ (or an LU factorisation) and recomputes quantities as needed. The Big-M method provides an alternative to two-phase simplex for finding an initial BFS.

Revised Simplex Method

Definition 4.3.1 — Revised Simplex

The revised simplex method maintains the current basis $B$ and the basis inverse $A_B^{-1}$ explicitly. Each iteration proceeds:

  1. Compute dual variables: $y^\top = c_B^\top A_B^{-1}$.
  2. Compute reduced costs for non-basic columns: $\bar{c}_j = c_j - y^\top A_j$ for $j \in N$.
  3. If $\bar{c}_j \ge 0$ for all $j \in N$: stop, current BFS is optimal.
  4. Select entering variable $q$ (e.g., most negative $\bar{c}_j$).
  5. Compute direction $d = A_B^{-1} A_q$ (one solve with $A_B$).
  6. Perform minimum ratio test to find leaving variable $p$.
  7. Update $B$: replace $p$ with $q$, update $A_B^{-1}$.

The critical operation is step 7: updating $A_B^{-1}$ after a rank-1 change to the basis.

Theorem 4.3.2 — Sherman-Morrison Rank-1 Update

If $A_B^{-1}$ is known and we replace column $p$ of $A_B$ with $A_q$, giving a new basis matrix $\tilde{A}_B$, then $$\tilde{A}_B^{-1} = A_B^{-1} - \frac{(A_B^{-1} A_q - e_p) e_p^\top A_B^{-1}}{e_p^\top A_B^{-1} A_q},$$ where $e_p$ is the $p$-th standard basis vector. This is the Sherman-Morrison formula for a rank-1 update.

In the simplex context, $d = A_B^{-1} A_q$ is already computed in step 5, so the update requires $O(m^2)$ operations — much cheaper than re-inverting $A_B$ from scratch ($O(m^3)$).

Remark — LU Factorisation for Numerical Stability

In practice, $A_B^{-1}$ is never stored or updated explicitly. Instead, an LU factorisation $A_B = LU$ (with partial pivoting) is maintained. Solves $A_B^{-1} v$ are performed via forward and backward substitution in $O(m^2)$ time.

After many iterations, accumulated rounding errors degrade the factorisation. Production solvers (CPLEX, Gurobi, HiGHS) refactorise $A_B$ periodically and use sophisticated data structures to exploit sparsity, which is crucial for large-scale LPs where $A$ may have fewer than $1\%$ nonzeros.

Big-M Method

Definition 4.3.3 — Big-M Method

The Big-M method augments the original LP with artificial variables $a_i \ge 0$ penalised by a large constant $M > 0$: $$\min\; c^\top x + M \sum_{i=1}^m a_i \quad\text{s.t.}\quad Ax + a = b,\; x, a \ge 0.$$ The initial BFS uses $a$ as the basis. If the optimal value has $a^* = 0$, we have solved the original LP. If $a^* \ne 0$ at optimality (for any finite $M$), the original LP is infeasible.

For sufficiently large $M$, the optimal solution of the penalised problem coincides with that of the original LP, provided the original is feasible.

Proposition 4.3.4 — Two-Phase vs Big-M

Both methods achieve the same goal: finding an initial BFS. Their tradeoffs are:

  • Two-phase: Cleanly separates feasibility from optimisation. Numerically more stable since Phase I costs are $O(1)$, not $O(M)$. Preferred in practice.
  • Big-M: Simpler to implement (single simplex run). However, very large $M$ causes numerical ill-conditioning: reduced costs of $O(M)$ drown out the true costs $O(1)$, leading to pivot selection errors. The "right" $M$ is problem-dependent.

For this reason, production solvers use two-phase (or related techniques such as column generation) rather than Big-M.

Example — Big-M on a Simple Problem

Minimise $x_1 + 2x_2$ s.t. $x_1 + x_2 \ge 3$, $x_1, x_2 \ge 0$. Convert: $x_1 + x_2 - s + a = 3$ with $s, a \ge 0$ (surplus $s$, artificial $a$). Big-M problem: $\min\; x_1 + 2x_2 + Ma$.

Initial BFS: $a = 3$, $x_1 = x_2 = s = 0$, objective $= 3M$. Reduced costs: $\bar{c}_{x_1} = 1 - M < 0$ for large $M$. Enter $x_1$; ratio test gives $x_1 = 3$, $a$ leaves. New BFS: $x_1 = 3$, $a = 0$. Reduced costs $\bar{c}_{x_2} = 1 \ge 0$, $\bar{c}_s = M \ge 0$. Optimal: $x_1^* = 3$, $x_2^* = 0$, objective $= 3$.

The artificial variable is driven to zero, confirming feasibility.
