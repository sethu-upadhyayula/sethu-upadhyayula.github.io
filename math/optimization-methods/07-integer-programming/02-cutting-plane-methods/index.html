7.2 Cutting Plane Methods

Cutting plane methods improve the LP relaxation by adding valid inequalities (cuts) that are satisfied by all integer feasible solutions but violated by the current fractional LP solution. Iterating this process tightens the LP bound until it equals the IP optimum.

Valid Inequalities

Definition 7.2.1 — Valid Inequality

An inequality $\pi^\top x \le \pi_0$ is valid for the IP if it is satisfied by all integer feasible solutions $x \in \{x : Ax \le b,\, x \ge 0,\, x_j \in \mathbb{Z}\;\forall j \in \mathcal{I}\}$.

A valid inequality is a cut (or cutting plane) if the current LP optimal solution $x^*_{\mathrm{LP}}$ violates it: $\pi^\top x^*_{\mathrm{LP}} > \pi_0$. Adding the cut to the LP eliminates $x^*_{\mathrm{LP}}$ and forces a new (tighter) LP optimal. The goal is to approximate the integer hull $\mathrm{conv}(\{x \in \mathcal{F}_{\mathrm{IP}}\})$.

Gomory Cuts

Theorem 7.2.2 — Gomory Fractional Cut

Let $x^*$ be the LP optimal BFS with basis $B$. Consider a basic variable $x_i = (A_B^{-1}b)_i = \bar{b}_i \notin \mathbb{Z}$. The $i$-th row of the optimal tableau is: $$x_i + \sum_{j \in N} \bar{a}_{ij} x_j = \bar{b}_i.$$ Write $\bar{b}_i = \lfloor\bar{b}_i\rfloor + f_i$ and $\bar{a}_{ij} = \lfloor\bar{a}_{ij}\rfloor + f_{ij}$ with fractional parts $f_i, f_{ij} \in [0,1)$. The Gomory cut is: $$\sum_{j \in N} f_{ij} x_j \ge f_i.$$

Validity: For any integer solution, $x_i + \sum_{j \in N} \bar{a}_{ij} x_j = \bar{b}_i \in \mathbb{Z}$, so $\sum_{j \in N} f_{ij} x_j - f_i \in \mathbb{Z}$. Since $x_j \ge 0$ and $f_{ij} \ge 0$, this integer is $\ge 0$, i.e., $\sum f_{ij} x_j \ge f_i$. $\square$

Violation at $x^*$: At $x^* = (x_B, 0)$, all non-basic $x_j = 0$, so LHS $= 0 < f_i$ (since $f_i > 0$). The cut is violated. Adding it eliminates $x^*$ from the LP.

Lift-and-Project and Cover Inequalities

Definition 7.2.3 — Cover Inequalities

For a 0-1 knapsack constraint $\sum_{j=1}^n a_j x_j \le b$ with $x_j \in \{0,1\}$, a set $C \subseteq \{1,\ldots,n\}$ is a cover if $\sum_{j \in C} a_j > b$ (the items in $C$ cannot all fit). The cover inequality is: $$\sum_{j \in C} x_j \le |C| - 1.$$

Cover inequalities are valid (at most $|C|-1$ items from $C$ can be selected) and often provide strong cuts for binary programmes. Lifted covers extend them to include variables not in $C$, yielding facet-defining inequalities.

Definition 7.2.4 — Lift-and-Project

The lift-and-project procedure (Balas, Ceria, Cornuéjols) generates cuts for binary variables by working in a higher-dimensional space. For a variable $x_j \in \{0,1\}$, multiply constraints by $x_j$ and $(1-x_j)$, forming the lifted set. Projecting back to the original space yields a cut that can be stronger than Gomory cuts.

Lift-and-project is equivalent to rank-1 Chvátal–Gomory cuts and provides a systematic way to generate the convex hull of mixed-integer feasible solutions for structured problems.

Branch-and-Cut

Definition 7.2.5 — Branch-and-Cut

Branch-and-cut combines branch-and-bound (Section 7.1) with cutting planes: at each B&B node, after solving the LP relaxation, attempt to add cuts before branching. If violated cuts are found, add them and re-solve the LP (tightening the bound without branching). Branch only when no more useful cuts can be found at a node.

The MIP solving pipeline in modern solvers (CPLEX, Gurobi, HiGHS) typically proceeds: presolve (remove redundancies, fix variables), add cuts at the root node (clique, cover, Gomory, MIR cuts), then branch-and-cut with heuristics for primal feasible solutions. This pipeline can solve MIPs with millions of variables.

Remark — Mixed-Integer Rounding (MIR) Cuts

Mixed-integer rounding (MIR) cuts generalise Gomory cuts to mixed-integer problems. For a constraint $\sum_j a_j x_j \le b$ with some integer and some continuous variables, the MIR cut is derived by rounding down coefficients and applying a similar fractional analysis.

MIR cuts, together with clique cuts (from the conflict graph of binary variables) and flow cover cuts (from network structure), are the most important classes in practice. The separation problem — finding the most violated cut of a given class — is itself an optimisation problem that must be solved efficiently within the B&C framework.
