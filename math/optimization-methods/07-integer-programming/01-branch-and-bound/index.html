7.1 Branch and Bound

Integer programming (IP) requires some or all decision variables to take integer values, making the feasible set non-convex and the problem NP-hard in general. Branch and bound (B&B) is the foundational exact algorithm: it systematically partitions the feasible set and uses LP relaxations as lower bounds to prune the search.

LP Relaxation

Definition 7.1.1 - Integer Programme and LP Relaxation

A mixed-integer linear programme (MILP) is $$\min\; c^\top x \quad\text{s.t.}\quad Ax \le b,\; x_j \in \mathbb{Z}\;\forall j \in \mathcal{I},\; x \ge 0,$$ where $\mathcal{I} \subseteq \{1,\ldots,n\}$ is the set of integer-constrained variables. The LP relaxation drops the integrality constraints: $\min c^\top x$ s.t. $Ax \le b$, $x \ge 0$.

Since the LP relaxation has a larger feasible set, its optimal value $z_{\mathrm{LP}}^*$ satisfies $z_{\mathrm{LP}}^* \le z_{\mathrm{IP}}^*$ (it is a lower bound). If the LP solution happens to be integer-valued, it is also IP-optimal.

Branch and Bound Algorithm

Definition 7.1.2 - Branch and Bound

The B&B algorithm maintains a search tree of subproblems. Each node corresponds to an LP with additional variable bounds.

Branching: If the LP relaxation at a node has a fractional solution $x_j^* \notin \mathbb{Z}$ for some $j \in \mathcal{I}$, create two child nodes by adding constraints: $$x_j \le \lfloor x_j^* \rfloor \quad\text{(down branch)},\qquad x_j \ge \lceil x_j^* \rceil \quad\text{(up branch)}.$$

Bounding: At each node, solve the LP relaxation. The LP objective value is a lower bound for that subtree.

Pruning: A node (subtree) is pruned if:

  • Its LP lower bound $\ge$ current best integer solution (incumbent).
  • Its LP is infeasible (no feasible integer solutions in subtree).
  • Its LP solution is integer-valued (update incumbent if better).

The algorithm terminates when all nodes are processed or pruned. The incumbent at termination is globally optimal.

Node Selection Strategies

Remark - Best-First vs Depth-First Search

The order in which nodes are processed affects efficiency:

  • Best-first search: Process the node with the smallest LP lower bound. Finds a proven optimal solution with fewer nodes but uses more memory (all active nodes must be stored).
  • Depth-first search: Process the most recently created node. Memory-efficient and quickly finds integer feasible solutions, but may explore more total nodes.
  • Best-first with plunging: Alternate between best-first and depth-first; used in practice. Plunging means following branches down to a leaf before backtracking.

Variable selection (branching): Most fractional (closest to 0.5), strong branching (evaluate both branches quickly), or pseudocost branching (estimate based on history). Branching heuristics have major practical impact.

Example - Simple B&B

Maximise $5x_1 + 4x_2$ s.t. $6x_1 + 4x_2 \le 24$, $x_1 + 2x_2 \le 6$, $x_1, x_2 \ge 0$, $x_1, x_2 \in \mathbb{Z}$.

Root LP: optimal at $x_1 = 3, x_2 = 1.5$, objective $= 21$. Branch on $x_2$:

  • Node 1 ($x_2 \le 1$): LP optimal $(3.33, 1)$, obj $= 20.67$. Branch on $x_1$: node with $x_1 \le 3$ gives integer solution $(3,1)$, obj $= 19$; node with $x_1 \ge 4$ gives $(4, 0)$, obj $= 20$. Incumbent $= 20$.
  • Node 2 ($x_2 \ge 2$): LP optimal $(2, 2)$, obj $= 18 < 20$. Prune.

Optimal: $x_1^* = 4, x_2^* = 0$, objective $= 20$.
