1.1 Addition and Multiplication Principles

The two most fundamental counting rules are the addition principle and the multiplication principle. Every combinatorial argument ultimately reduces to repeated applications of these two rules.

Theorem 1.1.1 — Addition Principle

If $A$ and $B$ are disjoint finite sets, then $|A \cup B| = |A| + |B|$. More generally, if $A_1, A_2, \ldots, A_k$ are pairwise disjoint finite sets, then $$|A_1 \cup A_2 \cup \cdots \cup A_k| = |A_1| + |A_2| + \cdots + |A_k|.$$

The addition principle applies when we are counting objects that fall into mutually exclusive categories: we count each category separately and add.

Example 1.1.1 — Passwords

How many strings of length 1 consist of either a digit (0–9) or an uppercase letter (A–Z)? The digits form a set $D$ of size 10; the letters form a set $L$ of size 26; since $D \cap L = \emptyset$, the total is $|D \cup L| = 10 + 26 = 36$.

How many strings of length 2 are there using these 36 characters? The addition principle does not directly apply here — we need the multiplication principle.

Theorem 1.1.2 — Multiplication Principle

If a procedure consists of $m$ steps, and step $i$ can be performed in $n_i$ ways regardless of how the earlier steps were performed, then the total number of ways to carry out the procedure is $$n_1 \cdot n_2 \cdots n_m.$$

Equivalently: if $A_1, A_2, \ldots, A_m$ are finite sets, then $|A_1 \times A_2 \times \cdots \times A_m| = |A_1| \cdot |A_2| \cdots |A_m|$.

Example 1.1.2 — Counting Functions

Let $A = \{1, 2, \ldots, m\}$ and $B = \{1, 2, \ldots, n\}$. A function $f: A \to B$ assigns to each element $i \in A$ some value $f(i) \in B$. There are $n$ choices for $f(1)$, then $n$ choices for $f(2)$, and so on, independently. By the multiplication principle, the total number of functions from $A$ to $B$ is $$n \cdot n \cdots n = n^m = |B|^{|A|}.$$

This formula is fundamental: it counts sequences of length $m$ over an alphabet of size $n$.

Example 1.1.3 — Subsets of $[n]$

How many subsets does $[n] = \{1, 2, \ldots, n\}$ have? Each subset $S \subseteq [n]$ corresponds to a function $f: [n] \to \{0,1\}$ (the indicator function $f(i) = 1$ iff $i \in S$). By the function-counting formula, the number of such functions is $2^n$.

Alternatively: for each of the $n$ elements, we independently decide whether to include it. The multiplication principle gives $\underbrace{2 \cdot 2 \cdots 2}_{n} = 2^n$ subsets.

Combining Both Principles

Example 1.1.4 — Passwords Revisited

A password consists of exactly 6 characters, where each character is either a digit (0–9) or an uppercase letter (A–Z), giving 36 choices per character. The total number of such passwords is $36^6 = 2{,}176{,}782{,}336$.

If we also allow passwords of length 5, 6, or 7, we compute the count for each length separately (using multiplication) and add the results (using addition): $$36^5 + 36^6 + 36^7 = 60{,}466{,}176 + 2{,}176{,}782{,}336 + 78{,}364{,}164{,}096 = 80{,}601{,}412{,}608.$$

This illustrates that addition and multiplication principles are used together repeatedly in counting arguments.

Remark — Independence

The multiplication principle requires that the number of choices at each step is independent of the previous steps. If the choices are not independent — for instance, if choosing an element at step $i$ removes it from the pool for step $i+1$ — we must count more carefully (this leads to permutations, Section 1.2).
