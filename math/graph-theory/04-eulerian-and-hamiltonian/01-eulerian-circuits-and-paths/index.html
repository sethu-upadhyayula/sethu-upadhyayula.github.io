4.1 Eulerian Circuits and Paths

In 1736, Leonhard Euler settled the question of the seven bridges of Königsberg, founding what we now call graph theory. The underlying question is purely combinatorial: can a graph be traversed by a closed walk that uses every edge exactly once? The answer turns out to depend only on the degrees of the vertices.

Definitions

Definition 4.1.1 - Eulerian Circuit and Eulerian Path

An Eulerian circuit (or Euler tour) of a graph $G$ is a closed walk that traverses every edge of $G$ exactly once (vertices may be repeated).

An Eulerian path (or Euler trail) is an open walk that traverses every edge exactly once but starts and ends at different vertices.

A graph is called Eulerian if it has an Eulerian circuit, and semi-Eulerian if it has an Eulerian path but not an Eulerian circuit.

Example 4.1.2 - The Seven Bridges of Königsberg

The city of Königsberg had two islands in the river Pregel, connected to each other and to the two banks by seven bridges. Euler modeled the four landmasses as vertices $A,B,C,D$ and the bridges as (multi-)edges between them. The question - is there a walk crossing each bridge exactly once and returning to the start? - is exactly the question of an Eulerian circuit.

In Euler's graph, every vertex has odd degree ($\deg A = \deg C = \deg D = 3$, $\deg B = 5$). Since more than two vertices have odd degree, by Theorem 4.1.4 (and even Theorem 4.1.6) no Eulerian circuit or path exists. This was Euler's negative resolution of the puzzle.

Euler's Theorem

Lemma 4.1.3 - Even Degree Necessary

If $G$ has an Eulerian circuit, then every vertex of $G$ has even degree. If $G$ has an Eulerian path between distinct vertices $u$ and $v$, then $u$ and $v$ have odd degree and every other vertex has even degree.

Proof

Each time the circuit (or path) passes through an internal vertex $w$, it uses one edge to enter and one to leave, contributing $2$ to $\deg(w)$. Over the whole traversal these contributions partition the edges incident to $w$ into pairs, so $\deg(w)$ is even - unless $w$ is an endpoint of an open trail, in which case one edge is left unpaired and $\deg(w)$ is odd. For a circuit, every vertex is "internal" (the start/end vertex is entered and left the same number of times), so all degrees are even. $\square$

Theorem 4.1.4 - Euler's Theorem (Circuits)

A connected graph $G$ (with at least one edge) has an Eulerian circuit if and only if every vertex of $G$ has even degree.

Proof

($\Rightarrow$) This is Lemma 4.1.3.

($\Leftarrow$) Suppose $G$ is connected and every vertex has even degree. We argue by induction on $|E(G)|$. Since every degree is even and $\ge 2$ (connectivity with $\ge 1$ edge forces $\delta(G) \ge 2$ when even and positive), $G$ contains a cycle $C$ - start at any vertex and keep walking along unused edges; since every vertex has even degree, whenever we enter a vertex (other than possibly the start) we can always leave, so the walk must eventually return to its starting vertex, forming a closed walk $C$ with no repeated edges.

Remove the edges of $C$ from $G$ to get $G' = G - E(C)$. Every vertex still has even degree in $G'$ (we removed $0$ or $2$ edges at each vertex of $C$). Each connected component of $G'$ with at least one edge is connected with all-even degrees and fewer edges than $G$, so by the induction hypothesis each has an Eulerian circuit. Moreover, each such component shares at least one vertex with $C$ (since $G$ is connected).

Splice these component circuits into $C$ at their shared vertices: walk along $C$, and whenever we reach a vertex shared with a component's circuit (not yet traversed), detour around that entire circuit before continuing along $C$. The result is a closed walk using every edge of $G$ exactly once - an Eulerian circuit. $\square$

Theorem 4.1.5 - Euler's Theorem (Paths)

A connected graph $G$ (with at least one edge) has an Eulerian path (and is not Eulerian) if and only if exactly two vertices of $G$ have odd degree. In this case, every Eulerian path must start at one odd-degree vertex and end at the other.

Proof Sketch

($\Rightarrow$) follows from Lemma 4.1.3, since exactly the two endpoints contribute an unpaired edge. ($\Leftarrow$) If $u,v$ are the two odd-degree vertices, add a new edge $\{u,v\}$ (or a path through a new vertex if $u,v$ already adjacent and the graph is simple) to form $G^+$, in which every vertex now has even degree and $G^+$ remains connected. By Theorem 4.1.4, $G^+$ has an Eulerian circuit; deleting the added edge from this circuit yields an Eulerian path of $G$ from $u$ to $v$. $\square$

Theorem 4.1.6 - Summary Criterion

Let $G$ be connected. Then:

  • $G$ has an Eulerian circuit $\iff$ the number of odd-degree vertices is $0$.
  • $G$ has an Eulerian path but no Eulerian circuit $\iff$ the number of odd-degree vertices is exactly $2$.
  • $G$ has neither $\iff$ the number of odd-degree vertices is $> 2$ (note this number is always even, by the handshake lemma).

Algorithms

Algorithm 4.1.7 - Fleury's Algorithm

Starting from a vertex of odd degree (or any vertex, if all degrees are even), repeatedly traverse an unused edge, choosing a bridge (cut edge) of the remaining unused-edge graph only if there is no alternative.

The "don't burn bridges unless forced to" rule ensures the remaining unused edges always stay connected to the current vertex, so the walk can always be extended until all edges are used. Fleury's algorithm is conceptually elegant but requires repeated bridge-detection, costing $O(E^2)$ in the naive implementation.

Algorithm 4.1.8 - Hierholzer's Algorithm

A linear-time $O(E)$ alternative:

  1. Starting from any vertex (an odd-degree vertex if seeking an Eulerian path), follow unused edges until returning to a vertex already on the current walk, forming a closed sub-tour.
  2. While some vertex on the current tour still has unused incident edges, splice in a new closed sub-tour starting and ending at that vertex (found the same way).
  3. Repeat until no unused edges remain. The resulting spliced walk is an Eulerian circuit (or path).

This is essentially the constructive proof of Theorem 4.1.4 turned into an algorithm, implemented efficiently with a stack.

Directed Graphs

Theorem 4.1.9 - Eulerian Digraphs

A connected digraph $D$ has a directed Eulerian circuit if and only if $\text{in-deg}(v) = \text{out-deg}(v)$ for every vertex $v$. It has a directed Eulerian path (start $\ne$ end) if and only if there is exactly one vertex $u$ with $\text{out-deg}(u) - \text{in-deg}(u) = 1$, exactly one vertex $v$ with $\text{in-deg}(v) - \text{out-deg}(v) = 1$, and all other vertices have equal in- and out-degree.

Applications

Remark - Applications

Eulerian circuits underlie the Chinese Postman Problem (route inspection problem): finding a minimum-length closed walk traversing every edge at least once, solved in polynomial time by adding minimum-weight duplicate edges to make all degrees even and then finding an Eulerian circuit. De Bruijn sequences, used in genome assembly and combinatorial generation, correspond to Eulerian circuits in de Bruijn graphs. Eulerian path criteria are also used in circuit board drilling and network traffic analysis.
