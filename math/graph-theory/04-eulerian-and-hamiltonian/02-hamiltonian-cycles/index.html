4.2 Hamiltonian Cycles

While Eulerian circuits visit every edge exactly once, Hamiltonian cycles visit every vertex exactly once. Despite the superficial similarity, the two problems could not be more different in difficulty: Eulerian circuits are characterized by a simple degree condition checkable in linear time, while no such characterization is known for Hamiltonian cycles - the decision problem is NP-complete.

Definitions

Definition 4.2.1 - Hamiltonian Path and Cycle

A Hamiltonian path in a graph $G$ is a path that visits every vertex of $G$ exactly once.

A Hamiltonian cycle is a cycle that visits every vertex of $G$ exactly once (and returns to its starting vertex).

A graph that contains a Hamiltonian cycle is called Hamiltonian.

Example 4.2.2 - Examples and Non-Examples

The complete graph $K_n$ ($n \ge 3$) is Hamiltonian: any ordering of its vertices forms a Hamiltonian cycle, since all vertices are pairwise adjacent.

The cycle $C_n$ is trivially Hamiltonian (it is a Hamiltonian cycle on itself).

The complete bipartite graph $K_{m,n}$ is Hamiltonian if and only if $m = n$ (a Hamiltonian cycle must alternate between the two parts, forcing equal sizes).

The Petersen graph is famously non-Hamiltonian - it has a Hamiltonian path but no Hamiltonian cycle, despite being $3$-regular, $3$-connected, and vertex-transitive. This makes it a key counterexample to many plausible-looking sufficient conditions.

Sufficient Conditions

Unlike the Eulerian case, no known condition is both necessary and sufficient for Hamiltonicity (short of an exhaustive search). However, several elegant sufficient conditions guarantee Hamiltonicity for "dense enough" graphs.

Theorem 4.2.3 - Dirac's Theorem (1952)

Let $G$ be a simple graph on $n \ge 3$ vertices. If every vertex has degree at least $n/2$, i.e. $$\delta(G) \ge \frac{n}{2},$$ then $G$ is Hamiltonian.

Proof Sketch

First, $G$ is connected: if not, the smaller component would have at most $\lfloor n/2 \rfloor$ vertices, but a vertex in it would need degree $\ge n/2$ within that component — impossible for a component with fewer than $n/2 + 1$ vertices.

Take a longest path $P = v_1 v_2 \cdots v_k$ in $G$. Since $P$ is longest, all neighbors of $v_1$ and $v_k$ lie on $P$ (otherwise $P$ could be extended). Both $v_1$ and $v_k$ have degree $\ge n/2$, so by a counting (pigeonhole) argument there exists some $i$ such that $v_1 v_{i+1} \in E$ and $v_k v_i \in E$. This produces a cycle $v_1 v_2 \cdots v_i v_k v_{k-1} \cdots v_{i+1} v_1$ on the same $k$ vertices.

If $k = n$, this cycle is already Hamiltonian. If $k < n$, since $G$ is connected there is a vertex outside the cycle adjacent to some vertex on it, which together with the cycle's structure allows extending to a longer path — contradicting that $P$ was longest. Hence $k = n$ and $G$ is Hamiltonian. $\square$

Theorem 4.2.4 - Ore's Theorem (1960)

Let $G$ be a simple graph on $n \ge 3$ vertices. If for every pair of non-adjacent vertices $u, v$, $$\deg(u) + \deg(v) \ge n,$$ then $G$ is Hamiltonian.

Remark - Ore Generalizes Dirac

Dirac's theorem follows from Ore's: if $\delta(G) \ge n/2$, then for any non-adjacent pair $u,v$ we have $\deg(u) + \deg(v) \ge n/2 + n/2 = n$. The proof of Ore's theorem uses essentially the same longest-path argument as Dirac's, applied to a non-adjacent pair $v_1, v_k$ at the ends of a longest path.

The Bondy–Chvátal Closure

Definition 4.2.5 - Closure of a Graph

The closure $\text{cl}(G)$ of a graph $G$ on $n$ vertices is obtained by repeatedly adding an edge between any two non-adjacent vertices $u,v$ satisfying $\deg(u) + \deg(v) \ge n$, until no such pair remains. (The result is independent of the order in which edges are added.)

Theorem 4.2.6 - Bondy–Chvátal Theorem (1976)

A graph $G$ on $n \ge 3$ vertices is Hamiltonian if and only if its closure $\text{cl}(G)$ is Hamiltonian. In particular, $G$ is Hamiltonian if $\text{cl}(G) = K_n$.

Remark - Strength of the Closure

The Bondy–Chvátal theorem strictly generalizes both Dirac's and Ore's theorems: if $\delta(G) \ge n/2$ (Dirac) or $G$ satisfies Ore's degree-sum condition for all non-adjacent pairs, then $\text{cl}(G) = K_n$ immediately, recovering Hamiltonicity. But there exist graphs whose closure is $K_n$ even though they satisfy neither Dirac's nor Ore's condition directly — the closure operation can "discover" Hamiltonicity that single-step degree conditions miss.

Computational Complexity

Theorem 4.2.7 - NP-Completeness

The decision problem "Does graph $G$ contain a Hamiltonian cycle?" is NP-complete (Karp, 1972). Consequently, no polynomial-time algorithm is known (and none is expected to exist unless $\mathrm{P} = \mathrm{NP}$) that decides Hamiltonicity for arbitrary graphs.

Remark - Eulerian vs. Hamiltonian

This is the central contrast of the chapter: Eulerian circuits (Section 4.1) admit a clean, efficiently checkable degree-based characterization (Theorem 4.1.4), and Hierholzer's algorithm finds one in linear time when it exists. Hamiltonian cycles have no known analogous characterization; the sufficient conditions of Dirac, Ore, and Bondy–Chvátal apply only to sufficiently dense graphs, and verifying Hamiltonicity in general remains computationally intractable. The Traveling Salesman Problem — finding a minimum-weight Hamiltonian cycle — is the most famous application and is itself NP-hard.
