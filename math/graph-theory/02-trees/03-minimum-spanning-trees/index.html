2.3 Minimum Spanning Trees

Given a connected weighted graph, a minimum spanning tree (MST) is a spanning tree whose total edge weight is as small as possible. MSTs model least-cost networks and arise in clustering, network design, and approximation algorithms.

Setup and Optimality Conditions

Definition 2.3.1 - Weighted Graph and MST

A weighted graph is a graph $G = (V, E)$ with a weight function $w : E \to \mathbb{R}$. The weight of a spanning tree $T$ is $w(T) = \sum_{e \in E(T)} w(e)$. A minimum spanning tree is a spanning tree $T^*$ minimising $w(T^*)$ over all spanning trees of $G$.

Theorem 2.3.2 - Cut Property

Let $(S, V \setminus S)$ be any cut of $G$ (partition of $V$ into two non-empty sets). If $e$ is a minimum-weight edge crossing the cut (with one endpoint in $S$ and one in $V \setminus S$), then $e$ belongs to some MST of $G$.

Theorem 2.3.3 - Cycle Property

If $e$ is a maximum-weight edge in some cycle $C$ of $G$ (and the maximum is unique), then $e$ does not belong to any MST of $G$.

Theorem 2.3.4 - Uniqueness

If all edge weights of $G$ are distinct, then $G$ has a unique MST.

Kruskal's Algorithm

Algorithm 2.3.5 - Kruskal (1956)

Sort all edges by non-decreasing weight. Initialise $T = \emptyset$. Process edges in order: add edge $e$ to $T$ if and only if $e$ does not create a cycle in $T$ (equivalently, its endpoints lie in different components of $T$).

Correctness follows from the cut property: each added edge is the minimum crossing some cut. Complexity: $O(E \log E)$ for sorting; union-find gives near-linear component queries, giving overall $O(E \log E) = O(E \log V)$.

Prim's Algorithm

Algorithm 2.3.6 - Prim (1957)

Start with any vertex $r$; let $S = \{r\}$ and $T = \emptyset$. Repeatedly find the minimum-weight edge $\{u, v\}$ with $u \in S$ and $v \notin S$; add $v$ to $S$ and $\{u,v\}$ to $T$. Repeat until $S = V$.

Correctness follows from the cut property applied to the cut $(S, V \setminus S)$. Complexity: $O(E \log V)$ with a binary heap; $O(E + V \log V)$ with a Fibonacci heap.

Borůvka's Algorithm

Algorithm 2.3.7 - Borůvka (1926)

Initially each vertex is its own component. In each phase, each component selects its cheapest outgoing edge; add all selected edges (merging components). Repeat until only one component remains.

After each phase the number of components halves, so there are $O(\log V)$ phases, each taking $O(E)$ time. Total: $O(E \log V)$. Borůvka's algorithm naturally parallelises and is used in external-memory and randomised MST algorithms.

Remark - Linear-Time MST

Combining Borůvka's algorithm with random edge contraction, Karger, Klein, and Tarjan (1995) gave a randomised linear-time MST algorithm. A deterministic linear-time algorithm (on a RAM) was given by Pettie and Ramachandran (2002), though it is not practical.
