2.1 Merge Sort and Binary Search

Merge sort and binary search are the canonical examples of divide and conquer. Merge sort is optimal among comparison-based sorting algorithms; binary search is optimal among comparison-based search in sorted arrays.

Merge Sort

Algorithm 2.1.1 - Merge Sort

To sort array $A[1..n]$: (1) if $n \le 1$, return; (2) split into $A[1..n/2]$ and $A[n/2+1..n]$; (3) sort each half recursively; (4) merge the two sorted halves in $O(n)$ time.

Recurrence: $T(n) = 2T(n/2) + \Theta(n) = \Theta(n \log n)$ by the Master Theorem (Case 2). Merge sort is stable (equal elements preserve relative order).

Theorem 2.1.2 - Comparison Sort Lower Bound

Any comparison-based sorting algorithm requires $\Omega(n \log n)$ comparisons in the worst case. Proof: the decision tree for $n$ elements has $n!$ leaves; the tree has height $\ge \log_2(n!) = \Omega(n \log n)$ by Stirling's approximation. Hence merge sort is asymptotically optimal.

Binary Search

Algorithm 2.1.3 - Binary Search

To find target $x$ in sorted array $A[1..n]$: compare $x$ with $A[\lfloor n/2 \rfloor]$; if equal, return; if $x < A[\lfloor n/2 \rfloor]$, recurse on left half; else recurse on right half.

Recurrence: $T(n) = T(n/2) + O(1) = O(\log n)$. Returns in $\lfloor \log_2 n \rfloor + 1$ comparisons.

Example 2.1.4 - Binary Search Variants

First occurrence: when target found, do not stop; continue in the left half to find the leftmost occurrence.

Search in rotated sorted array: determine which half is sorted; recurse on the appropriate half.

Exponential search: find range $[2^k, 2^{k+1}]$ containing target in $O(\log i)$ where $i$ is the position; then binary search within range.

Practical Variants

Remark - TimSort

Python's sort() and Java's Arrays.sort() use TimSort: a hybrid of merge sort and insertion sort. It exploits existing sorted runs in the data (natural merge sort) and uses insertion sort for small subarrays (typically $n < 64$). Worst-case $O(n \log n)$; best-case $O(n)$ for nearly-sorted input.
