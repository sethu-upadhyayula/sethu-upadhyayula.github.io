2.3 Closest Pair of Points

Given $n$ points in the Euclidean plane $\mathbb{R}^2$, the closest pair problem asks for the pair with minimum Euclidean distance. The brute-force $O(n^2)$ approach checks all $\binom{n}{2}$ pairs; divide and conquer achieves $O(n \log n)$.

Theorem 2.3.1 — Closest Pair via Divide and Conquer

The following algorithm runs in $O(n \log n)$:

  1. Presort points by $x$-coordinate: $O(n \log n)$ once.
  2. Split into left half $L$ and right half $R$ at median $x$-coordinate.
  3. Recursively find $\delta_L = $ min dist in $L$ and $\delta_R = $ min dist in $R$. Set $\delta = \min(\delta_L, \delta_R)$.
  4. Consider the strip $|x - x_{\text{mid}}| \le \delta$. Sort strip points by $y$-coordinate.
  5. For each strip point, check at most 7 subsequent points (within distance $2\delta$ in $y$). Update $\delta$ if closer pair found.

The key insight: in a $\delta \times 2\delta$ rectangle, at most 8 points from each half (since all such points are $\ge \delta$ apart within each half). This bounds the inner loop to $O(1)$ per strip point.

Theorem 2.3.2 — Time Complexity

With presorting by $x$ once, the strip is re-sorted by $y$ at each level: $T(n) = 2T(n/2) + O(n \log n) = O(n \log^2 n)$. With the merge trick (maintain $y$-sorted order during the merge step, like merge sort), the strip sorting at each level is $O(n)$, giving $T(n) = 2T(n/2) + O(n) = O(n \log n)$.

Remark — Generalisation

The same divide-and-conquer idea extends to $\mathbb{R}^d$ for any fixed $d$: runtime $O(n \log n)$ with the merge trick (the constant in the strip check grows with $d$ but remains bounded for fixed $d$). Applications include collision detection, clustering algorithms, and computational geometry.
