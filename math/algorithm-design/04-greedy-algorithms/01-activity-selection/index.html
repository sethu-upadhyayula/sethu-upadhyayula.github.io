4.1 Activity Selection

The activity selection problem is a scheduling problem that illustrates the greedy paradigm perfectly. A single resource can handle at most one activity at a time; we want to schedule the most activities possible.

Definition 4.1.1 - Activity Selection Problem

Given $n$ activities each with start time $s_i$ and finish time $f_i$ (with $s_i < f_i$), two activities $i$ and $j$ are compatible if their intervals $[s_i, f_i)$ and $[s_j, f_j)$ do not overlap (i.e., $f_i \le s_j$ or $f_j \le s_i$). Find a maximum-cardinality set of mutually compatible activities.

Theorem 4.1.2 - Greedy Activity Selection

Algorithm: Sort activities by finish time $f_1 \le f_2 \le \cdots \le f_n$. Initialise $A = \{1\}$ and $k = 1$ (last selected). For $i = 2, \ldots, n$: if $s_i \ge f_k$, add $i$ to $A$ and set $k = i$.

This runs in $O(n \log n)$ (dominated by sorting). The greedy choice of "earliest finish time" is optimal.

Proof of Correctness (Exchange Argument)

Let $G = \{g_1, \ldots, g_k\}$ (greedy solution, sorted by finish time) and $O = \{o_1, \ldots, o_m\}$ (any optimal solution, sorted). We claim $k = m$.

By induction: $f_{g_i} \le f_{o_i}$ for all $i$ (the greedy choice finishes no later than any other). If $k < m$, after the last greedy selection $g_k$ there would be activities in $O$ compatible with $g_k$ (since $f_{g_k} \le f_{o_k} \le s_{o_{k+1}}$), contradicting the greedy selection stopping. $\square$

Variants and Limitations

Example 4.1.3 - Weighted Job Scheduling

If activities have weights (profits) and we want to maximise total weight, the greedy approach fails. Dynamic programming is needed: $\mathrm{dp}[i]$ = max profit using activities from $\{1,\ldots,i\}$ (sorted by finish time); $\mathrm{dp}[i] = \max(\mathrm{dp}[i-1],\, w_i + \mathrm{dp}[p(i)])$ where $p(i)$ is the last compatible activity before $i$. Time: $O(n \log n)$.

Example 4.1.4 - Interval Partitioning

Given $n$ lectures with time intervals, schedule all lectures using the minimum number of classrooms (no two in the same room at the same time). Greedy: sort by start time, assign each lecture to any available room; if none, open a new one. The minimum number of classrooms equals the maximum number of simultaneous lectures (depth). Implementable with a priority queue in $O(n \log n)$.
