7.1 P and NP

Complexity classes formalise the intuitive notion of "easy" vs "hard" problems. The classes $P$ and $NP$ are defined in terms of a computational model (Turing machines); the definitions are robust to reasonable model changes.

Definition 7.1.1 — The Class P

$P$ (polynomial time) is the class of decision problems (yes/no answer) that can be solved by a deterministic Turing machine in $O(n^k)$ time for some constant $k$, where $n$ is the input size. Examples: sorting (is array sorted?), graph connectivity, primality (AKS algorithm, 2002), 2-SAT, bipartite matching.

Definition 7.1.2 — The Class NP

$NP$ (nondeterministic polynomial time) is the class of decision problems for which a "yes" instance has a certificate (witness) that can be verified in polynomial time.

Equivalently: solvable by a nondeterministic Turing machine in polynomial time.

Examples: 3-SAT (certificate: a satisfying assignment), Hamiltonian cycle (certificate: the cycle), subset sum (certificate: the subset), graph $k$-colouring for $k \ge 3$ (certificate: the colouring).

Theorem 7.1.3 — $P \subseteq NP$

Every problem in $P$ is also in $NP$: given a "yes" instance, the algorithm itself serves as verifier (no certificate needed). Whether $P = NP$ is the Millennium Prize Problem (Clay Mathematics Institute, $1 million prize). The overwhelming consensus is $P \ne NP$, but no proof exists.

co-NP and the Complexity Hierarchy

Definition 7.1.4 — co-NP

$\text{co-NP}$ is the class of problems whose complement is in $NP$: "no" instances have polynomial-size certificates. Example: TAUTOLOGY (is every assignment satisfying?), graph non-Hamiltonicity.

PRIMES (is $n$ prime?) is in $P \cap \text{NP} \cap \text{co-NP}$ (AKS gave the P membership in 2002; it was in co-NP classically via Pratt certificates, and in NP via compositeness witnesses). It is believed $NP \ne \text{co-NP}$, which would imply $P \ne NP$.

Definition 7.1.5 — NP-Hard and NP-Complete

A problem $X$ is NP-hard if every problem in $NP$ can be reduced to $X$ in polynomial time (i.e., $X$ is at least as hard as the hardest NP problem). $X$ may not be a decision problem, and need not be in $NP$.

$X$ is NP-complete if $X \in NP$ and $X$ is NP-hard. NP-complete problems are the hardest problems in NP. If any NP-complete problem is in $P$, then $P = NP$.

Example 7.1.6 — NP Problems Landscape

Known to be in $P$: shortest paths, MST, network flow, 2-SAT, linear programming. Known NP-complete: 3-SAT, independent set, vertex cover, clique, graph 3-colouring, Hamiltonian cycle, TSP (decision), subset sum, knapsack (decision). Believed to be in $NP \cap \text{co-NP}$ but not known to be in $P$: factoring integers (the basis of RSA security).
