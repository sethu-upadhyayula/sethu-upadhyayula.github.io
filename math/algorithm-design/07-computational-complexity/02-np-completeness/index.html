7.2 NP-Completeness

NP-completeness identifies the "hardest" problems within $NP$: a problem $X$ is NP-complete if it is in $NP$ and every other $NP$ problem can be transformed into $X$ efficiently. The Cook-Levin theorem (1971) showed that such problems exist by exhibiting one explicitly — Boolean satisfiability (SAT) — and a vast catalogue of natural problems has since been shown NP-complete via reductions from it.

Polynomial-Time Reductions

Definition 7.2.1 — Polynomial-Time (Karp) Reduction

A language $A$ polynomial-time reduces to a language $B$, written $A \le_p B$, if there is a function $f$ computable in polynomial time such that

$$x \in A \iff f(x) \in B.$$

Intuitively: solving $B$ in polynomial time would let us solve $A$ in polynomial time (just transform the instance and call the $B$-solver). So $A \le_p B$ means "$A$ is no harder than $B$."

Remark — Cook vs. Karp Reductions

A Karp reduction (above) is a single many-one mapping of instances. A Cook (Turing) reduction is more general: $A$ is solved using polynomially many calls to an oracle for $B$, possibly with post-processing. Karp reductions are the standard tool for proving NP-completeness because they preserve membership in $NP$ directly; Cook reductions are used more broadly (e.g., for NP-hardness of search/optimisation problems that may not themselves be in $NP$).

Theorem 7.2.2 — Transitivity and Consequences

If $A \le_p B$ and $B \le_p C$, then $A \le_p C$ (compose the polynomial-time reductions).

If $A \le_p B$ and $B \in P$, then $A \in P$.

Contrapositive: if $A \le_p B$ and $A$ is NP-hard, then $B$ is NP-hard. This is the key tool for proving new problems NP-hard — reduce a known NP-hard problem to the new problem.

The Cook-Levin Theorem

Definition 7.2.3 — SAT and CNF

A Boolean formula is in conjunctive normal form (CNF) if it is an AND of clauses, each clause an OR of literals (variables or their negations), e.g.

$$(x_1 \lor \lnot x_2 \lor x_3) \land (\lnot x_1 \lor x_2) \land (x_2 \lor x_3 \lor \lnot x_4).$$

SAT: given a CNF formula $\varphi$, does there exist an assignment of truth values to the variables making $\varphi$ true? $k$-SAT is SAT restricted to formulas where every clause has exactly $k$ literals.

Theorem 7.2.4 — Cook-Levin Theorem

SAT is NP-complete.

Proof Sketch

SAT $\in NP$: a satisfying assignment is a certificate of polynomial size; checking that it satisfies $\varphi$ takes polynomial time.

SAT is NP-hard: let $L$ be any language in $NP$, decided by a nondeterministic Turing machine $M$ in time $p(n)$ for a polynomial $p$. We construct, for each input $x$ of length $n$, a CNF formula $\varphi_x$ of size $\text{poly}(n)$ such that $\varphi_x$ is satisfiable iff $M$ accepts $x$.

The formula encodes the entire computation history of $M$ on $x$ as a $p(n) \times p(n)$ "tableau": cell $(i,t)$ holds the tape symbol/state/head-position information at time step $t$, position $i$. Boolean variables represent the contents of each cell, and clauses enforce:

(1) Cell consistency — each cell holds exactly one symbol/state combination; (2) Start configuration — row $0$ encodes $x$ and $M$'s initial state; (3) Transition validity — each $2\times3$ window of consecutive cells is consistent with $M$'s transition function (this is where nondeterminism is captured — multiple transitions are allowed as alternatives within a clause); (4) Acceptance — some cell in the final row is in an accepting state.

Each constraint translates into $O(1)$ clauses per tableau cell, giving $O(p(n)^2)$ clauses overall — polynomial in $n$. The construction itself runs in polynomial time. Hence $L \le_p \text{SAT}$ for every $L \in NP$, so SAT is NP-hard, and combined with SAT $\in NP$, SAT is NP-complete. $\blacksquare$

Remark — Why This Matters

Cook-Levin gives us one NP-complete problem "from scratch," via direct simulation of Turing machines. Every subsequent NP-completeness proof can avoid this machine-level argument: to show $X$ is NP-hard, it suffices to exhibit $Y \le_p X$ for some already-known NP-hard $Y$ (often SAT or 3-SAT), by Theorem 7.2.2.

3-SAT and the First Reductions

Theorem 7.2.5 — 3-SAT is NP-complete

SAT $\le_p$ 3-SAT, so 3-SAT is NP-hard; and 3-SAT $\in NP$ (it is a special case of SAT), so 3-SAT is NP-complete. (Note: 2-SAT is in $P$, solvable via strongly connected components of the implication graph — so the jump from 2 to 3 literals per clause crosses the $P$/$NP$-complete boundary.)

Proof Idea — SAT $\le_p$ 3-SAT

Convert each clause of $\varphi$ to one or more clauses with exactly 3 literals:

A clause with 1 literal $(\ell_1)$: introduce two fresh variables $p, q$ and replace by $(\ell_1 \lor p \lor q) \land (\ell_1 \lor \lnot p \lor q) \land (\ell_1 \lor p \lor \lnot q) \land (\ell_1 \lor \lnot p \lor \lnot q)$ — satisfiable in $\ell_1$ regardless of $p,q$ iff $\ell_1$ true.

A clause with 2 literals $(\ell_1 \lor \ell_2)$: add a fresh variable $p$, replace by $(\ell_1 \lor \ell_2 \lor p) \land (\ell_1 \lor \ell_2 \lor \lnot p)$.

A clause with $k > 3$ literals $(\ell_1 \lor \cdots \lor \ell_k)$: introduce fresh variables $z_1, \ldots, z_{k-3}$ and chain: $(\ell_1 \lor \ell_2 \lor z_1) \land (\lnot z_1 \lor \ell_3 \lor z_2) \land \cdots \land (\lnot z_{k-3} \lor \ell_{k-1} \lor \ell_k)$. Each transformation preserves satisfiability and increases formula size by only a constant factor, so the whole reduction is polynomial-time. $\blacksquare$

A Gallery of NP-Complete Problems

Example 7.2.6 — Vertex Cover (3-SAT $\le_p$ Vertex Cover)

Vertex Cover: given a graph $G = (V,E)$ and integer $k$, does $G$ have a set $S \subseteq V$ with $|S| \le k$ such that every edge has at least one endpoint in $S$?

Reduction from 3-SAT: for a formula with variables $x_1,\ldots,x_n$ and clauses $C_1,\ldots,C_m$ (each with 3 literals), build a graph with:

(a) A variable gadget for each $x_i$: an edge between nodes labelled $x_i$ and $\lnot x_i$ — covering this edge forces choosing at least one of "$x_i$ true" or "$x_i$ false".

(b) A clause gadget for each clause $C_j$: a triangle on three nodes representing its three literals — covering a triangle's 3 edges requires at least 2 of its 3 vertices.

(c) Connect each clause-gadget literal vertex to the corresponding variable-gadget vertex with the same literal.

Set $k = n + 2m$. One can show $\varphi$ is satisfiable iff $G$ has a vertex cover of size $k$: the $n$ variable-edge picks encode the assignment, and the $2m$ triangle picks can always cover the "satisfied" literal's connecting edge for each clause iff the assignment satisfies $\varphi$. Since 3-SAT $\le_p$ Vertex Cover and Vertex Cover $\in NP$ (certificate: the cover $S$), Vertex Cover is NP-complete.

Example 7.2.7 — Clique and Independent Set

Clique: does $G$ contain a complete subgraph on $k$ vertices? Independent Set: does $G$ contain $k$ pairwise non-adjacent vertices?

$S$ is a clique in $G$ iff $S$ is an independent set in the complement graph $\bar G$, and $S$ is a vertex cover of $G$ iff $V \setminus S$ is an independent set in $G$. These trivial transformations give Vertex Cover $\le_p$ Independent Set $\le_p$ Clique (and back), so all three are NP-complete (each is clearly in $NP$: the witness is the vertex set itself).

Example 7.2.8 — Hamiltonian Cycle

Hamiltonian Cycle: does $G$ have a cycle visiting every vertex exactly once?

In $NP$: the cycle itself (a permutation of $V$) is a polynomial-size certificate, checkable in $O(|V|)$ time. NP-hardness is shown by a (more involved) reduction from 3-SAT or vertex cover, using "gadgets" — small subgraphs that force a Hamiltonian cycle to pass through them in a pattern that encodes a variable assignment or a clause-satisfaction choice. The decision version of the Travelling Salesman Problem ("is there a tour of total weight $\le k$?") is NP-complete via a direct reduction from Hamiltonian Cycle: set edge weights $1$ for graph edges and $2$ for non-edges, and ask for a tour of weight $\le |V|$.

Example 7.2.9 — Subset Sum

Subset Sum: given positive integers $a_1, \ldots, a_n$ and target $T$, does some subset sum to exactly $T$?

In $NP$: the subset itself is the certificate. NP-hardness via reduction from 3-SAT (or from the "Exact Cover" family): each variable and clause is encoded as a decimal digit position in a large number, with values chosen so that a subset summing to $T$ corresponds to a satisfying assignment. Subset Sum is solvable in pseudo-polynomial time $O(nT)$ via dynamic programming (Section 3.3) — this does not contradict NP-hardness because $T$ can be exponential in the input's bit-length $\log T$.

Proving a New Problem NP-Complete

Recipe — The Standard Template

To show a decision problem $X$ is NP-complete:

1. $X \in NP$: exhibit a certificate format and a polynomial-time verifier.

2. $X$ is NP-hard: choose a known NP-complete problem $Y$ (often 3-SAT, Vertex Cover, or another problem structurally close to $X$), and construct a polynomial-time function $f$ mapping instances of $Y$ to instances of $X$ such that $y$ is a "yes" instance of $Y$ iff $f(y)$ is a "yes" instance of $X$.

3. Prove correctness of the reduction in both directions ("$\Rightarrow$": a solution to $y$ gives a solution to $f(y)$; "$\Leftarrow$": a solution to $f(y)$ gives a solution to $y$), and check $f$ runs in polynomial time.

Worked Example 7.2.10 — Graph 3-Colouring is NP-Complete

Problem: given $G = (V,E)$, can each vertex be coloured with one of 3 colours so that adjacent vertices differ?

Step 1 ($\in NP$): a colouring $c: V \to \{1,2,3\}$ is the certificate; checking $c(u) \ne c(v)$ for every edge $(u,v)$ takes $O(|E|)$ time.

Step 2 (reduction from 3-SAT): given a 3-SAT formula $\varphi$ with variables $x_1,\ldots,x_n$ and clauses $C_1,\ldots,C_m$, build $G$ as follows. Use 3 special "palette" vertices $T$ (true), $F$ (false), $B$ (base), forming a triangle (forcing them to receive 3 distinct colours, fixed as labels "true"/"false"/"base"). For each variable $x_i$, add vertices $x_i, \overline{x_i}$, each connected to $B$ and to each other — forcing $\{x_i, \overline{x_i}\}$ to take colours $\{T,F\}$ in some order, which encodes the truth assignment.

For each clause $C_j = (\ell_a \lor \ell_b \lor \ell_c)$, add a small "OR gadget" of 5 auxiliary vertices connected to $B$, $T$, and the three literal vertices $\ell_a,\ell_b,\ell_c$, designed so that the gadget can be properly 3-coloured iff at least one of $\ell_a,\ell_b,\ell_c$ is coloured $T$ (i.e., at least one literal in $C_j$ is true).

Step 3 (correctness): if $\varphi$ is satisfiable, colour $x_i$ by $T$ or $F$ according to the satisfying assignment (and $\overline{x_i}$ the opposite); each clause gadget can then be completed validly since some literal is true. Conversely, a proper 3-colouring of $G$ assigns each $x_i$ a colour in $\{T,F\}$ (forced by the $B$-triangle gadget), and the clause-gadget colourability constraint forces at least one literal per clause to be $T$ — giving a satisfying assignment. The graph has $O(n+m)$ vertices and edges, built in polynomial time. Hence 3-SAT $\le_p$ 3-Colouring, so 3-Colouring is NP-complete.

Remark — The Web of Reductions

In practice one rarely reduces from SAT directly; instead one picks the "closest" known NP-complete problem to minimise the reduction's complexity. Common hub problems include 3-SAT, Vertex Cover/Independent Set/Clique, Hamiltonian Cycle/TSP, and 3-Colouring — together these cover most reductions encountered in scheduling, packing, partitioning, and graph-layout problems.
