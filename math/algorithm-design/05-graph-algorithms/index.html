Chapter 5: Graph Algorithms

Graph algorithms solve fundamental problems — reachability, shortest paths, and topological ordering — in near-linear time. Graphs model relationships ubiquitously: road networks, social networks, dependency graphs, and circuit layouts all admit graph-algorithmic treatment.

The two foundational traversal strategies, breadth-first search and depth-first search, underpin nearly every graph algorithm. From them we derive shortest-path algorithms, strongly connected components, and topological orderings.

Chapter Overview

5.1 — BFS and DFS

Breadth-first search explores vertices level by level using a FIFO queue, computing unweighted shortest paths in $O(V+E)$. Depth-first search uses a stack (or recursion) and produces DFS trees whose back edges detect cycles; Tarjan's and Kosaraju's algorithms find strongly connected components in $O(V+E)$.

5.2 — Shortest Paths

Dijkstra's algorithm handles non-negative edge weights in $O((V+E)\log V)$ via a min-heap. Bellman-Ford tolerates negative weights in $O(VE)$ and detects negative cycles. Floyd-Warshall computes all-pairs shortest paths in $O(V^3)$. Johnson's algorithm combines both for sparse graphs.

5.3 — Topological Sorting

A topological order of a DAG is a linear ordering of vertices such that every directed edge $(u,v)$ has $u$ before $v$. Both DFS-based and Kahn's (in-degree queue) algorithms produce a topological order in $O(V+E)$, and detect cycles when none exists.
