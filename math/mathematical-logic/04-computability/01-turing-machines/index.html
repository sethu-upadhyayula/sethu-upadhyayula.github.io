4.1 Turing Machines

A Turing machine is an abstract model of computation introduced by Alan Turing in 1936. It formalises the intuitive notion of an algorithm and defines the boundary of what can be computed mechanically.

Definition 4.1.1 — Turing Machine

A Turing machine is a tuple $M = (Q, \Sigma, \Gamma, \delta, q_0, q_{\text{acc}}, q_{\text{rej}})$ where:

  • $Q$ is a finite set of states;
  • $\Sigma$ is the input alphabet (not containing the blank symbol $\sqcup$);
  • $\Gamma \supseteq \Sigma \cup \{\sqcup\}$ is the tape alphabet;
  • $\delta: Q \times \Gamma \to Q \times \Gamma \times \{L, R\}$ is the transition function;
  • $q_0 \in Q$ is the start state;
  • $q_{\text{acc}}, q_{\text{rej}} \in Q$ are the accept and reject states.

The tape is infinite in both directions and initially contains the input string with blanks everywhere else. The read-write head starts at the leftmost input symbol.

Computation

Definition 4.1.2 — Step of Computation

At each step, the machine reads the symbol $a \in \Gamma$ under the head. Given current state $q$, it applies $\delta(q, a) = (q', b, D)$:

  • Write symbol $b$ to the current tape cell;
  • Move the head left ($D = L$) or right ($D = R$);
  • Transition to state $q'$.

Computation halts when the machine enters $q_{\text{acc}}$ (accept) or $q_{\text{rej}}$ (reject). If neither is reached, the machine loops indefinitely.

Remark — Three Outcomes

On any input $w$, a TM $M$ either: (1) accepts $w$ (enters $q_{\text{acc}}$), (2) rejects $w$ (enters $q_{\text{rej}}$), or (3) loops forever. A TM that always halts (never loops) is called a decider.

Church-Turing Thesis

Church-Turing Thesis

Every effectively computable function — any function that can be computed by a systematic, finite procedure — is computable by a Turing machine. This is not a theorem (it cannot be formally proved) but is universally accepted based on the equivalence of all known models of computation.

Equivalent Models

Equivalent Models of Computation

The following models are all equivalent in computational power to the standard single-tape TM:

  • Multi-tape TMs: same power; $k$ tapes can be simulated by one tape with at most quadratic overhead in time;
  • Non-deterministic TMs (NTMs): same power; an NTM accepting in $t$ steps can be simulated deterministically in $O(2^t)$ steps;
  • $\lambda$-calculus (Church, 1936): functions defined by $\beta$-reduction;
  • $\mu$-recursive functions (Kleene): built from primitive recursion and minimisation.

The equivalence of these diverse models strongly supports the Church-Turing thesis.

The Universal Turing Machine

Theorem 4.1.1 — Universal Turing Machine

There exists a TM $U$ (the Universal Turing Machine) that, given the encoding $\langle M, w \rangle$ of a TM $M$ and an input string $w$, simulates the computation of $M$ on $w$:

$$U(\langle M, w \rangle) \text{ accepts} \iff M(w) \text{ accepts.}$$

The UTM is fundamental: it shows that a single fixed machine can carry out any computation, given an appropriate description of the desired computation. This is the theoretical basis for stored-program computers.

Remark — Encoding

We can encode any TM $M$ as a string $\langle M \rangle$ over a fixed alphabet. This encoding (Gödel numbering applied to TMs) allows TMs to reason about other TMs — a key technique in the proofs of undecidability and Gödel's theorems.
