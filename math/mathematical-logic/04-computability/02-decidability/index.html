4.2 Decidability

Decidability distinguishes problems that can be solved algorithmically (in all cases, always halting) from those that cannot. This is one of the deepest results in the foundations of mathematics and computer science.

Definition 4.2.1 - Decidable and Recognisable Languages

Decidable (recursive): A language $L$ is decidable if there exists a TM $M$ that halts on every input and accepts iff the input is in $L$. Such a TM is called a decider.

Turing-recognisable (recursively enumerable): A language $L$ is recognisable if there exists a TM $M$ that accepts every $w \in L$ and either rejects or loops on $w \notin L$.

Theorem 4.2.1 - Complement Characterisation

A language $L$ is decidable if and only if both $L$ and its complement $\bar{L}$ are Turing-recognisable. In particular, if $L$ is decidable then $\bar{L}$ is also decidable.

The Halting Problem

Definition 4.2.2 - Halting Problem

The Halting Problem is the language $$HP = \{\langle M, w \rangle : M \text{ is a TM that halts on input } w\}.$$ It asks: given a TM $M$ and input $w$, does $M$ eventually halt?

Theorem 4.2.2 - Undecidability of the Halting Problem

$HP$ is Turing-recognisable but not decidable.

Proof - Diagonalisation

Suppose for contradiction that a decider $H$ for $HP$ exists. Construct TM $D$ as follows:

$D(\langle M \rangle)$: run $H(\langle M, \langle M \rangle\rangle)$; if $H$ accepts (i.e., $M$ halts on $\langle M \rangle$), then loop forever; if $H$ rejects, then accept.

Now consider $D(\langle D \rangle)$:

  • If $D$ halts on $\langle D \rangle$: then $H$ accepts $\langle D, \langle D \rangle\rangle$, so $D$ loops - contradiction.
  • If $D$ does not halt on $\langle D \rangle$: then $H$ rejects $\langle D, \langle D \rangle\rangle$, so $D$ accepts (halts) - contradiction.

In both cases we have a contradiction. Therefore no such $H$ exists. $\square$

Rice's Theorem

Theorem 4.2.3 - Rice's Theorem

Let $P$ be any property of Turing-recognisable languages that is non-trivial (holds for some TMs and fails for others). Then the language $$L_P = \{\langle M \rangle : \mathcal{L}(M) \text{ has property } P\}$$ is undecidable.

In other words, no algorithm can decide any non-trivial semantic property of TMs - such as "does this TM accept the empty string?" or "does this TM recognise a regular language?"

Reductions

Definition 4.2.3 - Many-One Reduction

A language $A$ is mapping-reducible (many-one reducible) to $B$, written $A \le_m B$, if there exists a computable function $f$ such that for all $w$: $$w \in A \iff f(w) \in B.$$

Consequence: If $A \le_m B$ and $B$ is decidable, then $A$ is decidable. Contrapositively: if $A \le_m B$ and $A$ is undecidable, then $B$ is undecidable. Reductions are the primary tool for proving undecidability of new problems.

Example - $A_{TM}$ is Undecidable

The acceptance problem $A_{TM} = \{\langle M, w\rangle : M \text{ accepts } w\}$ is undecidable (by reduction from $HP$). It is Turing-recognisable: simply run $M$ on $w$ and accept if it does. But its complement is not recognisable, confirming $A_{TM}$ is not decidable.
