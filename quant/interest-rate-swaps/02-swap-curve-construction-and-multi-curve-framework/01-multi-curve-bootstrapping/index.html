2.1 Multi-Curve Bootstrapping

Section 1.2 established that modern swap valuation requires (at least) two curves: a discounting curve, built from OIS/SOFR instruments, and one or more forecasting curves, used to project the floating rates of each leg. This section covers how those curves are actually built — in sequence, with the discounting curve constructed first and forecast curves bootstrapped afterward, taking the discount curve as given.

Separating the Two Roles

The key conceptual move of the multi-curve framework is to stop treating "the yield curve" as a single object that simultaneously discounts cash flows and generates the forward rates used to project floating payments. Instead, these are two separate curves, calibrated to two separate sets of instruments.

Definition — Discount Curve and Forecast Curve

Let $P_d(0,T)$ denote the discount curve, used to discount the cash flows of any collateralized trade regardless of which index it references — in practice this is the SOFR/OIS curve, bootstrapped from OIS swap rates as in the single-curve methods of Section 2.1 of Rates Fundamentals. Let $P_x(0,T)$ denote a forecast curve for tenor $x$ (e.g. 3-month or 6-month), used only to generate forward rates $F_x(0;t_{i-1},t_i)$ via the usual relation $1+\delta_i F_x = P_x(0,t_{i-1})/P_x(0,t_i)$. The forecast curve is not used to discount cash flows — discounting always uses $P_d$.

Because $P_x$ is purely a device for generating forward rates and is never used for discounting, it does not need to represent "true" discount factors at all — it is, in effect, a bookkeeping curve whose only role is to reproduce the correct forward rates for tenor $x$ when fed through the forward-rate formula.

Bootstrapping Order: Discount First, Then Forecast

Because the value of every swap depends on $P_d$ (through discounting) and on the relevant $P_x$ (through the floating leg's forward rates), and because $P_d$ does not depend on any $P_x$, the natural and standard approach is sequential.

Remark — The Sequential Bootstrap

Step 1 — Bootstrap $P_d$. Use OIS/SOFR swap rates (which reference the same overnight rate used for both discounting and the floating leg) to bootstrap the discount curve exactly as in the single-curve case of Rates Fundamentals Section 2.1 — single-curve telescoping applies because the discounting rate and the floating reference rate coincide for OIS instruments.

Step 2 — Bootstrap each $P_x$ given $P_d$. For each tenor $x$ (e.g. 3M, 6M, 1M), take the par swap rates for fixed-vs-$x$ swaps and solve, period by period, for the forward rates $F_x$ that reprice each instrument given the now-known $P_d$. Build $P_x$ from those forward rates via the forward-rate relation in the definition above, anchored at $P_x(0,0)=1$.

Step 3 — Bootstrap tenor basis curves (Section 2.2). Where forecast curves for different tenors of the same currency are not independently observable from vanilla swaps alone, tenor basis swap quotes pin down the spread between, e.g., the 3M and 6M forecast curves.

Bootstrapping a Forecast Curve Given the Discount Curve

The forecast-curve bootstrap follows the same node-by-node logic as a single-curve bootstrap, but the par-rate equation now mixes two curves: the annuity uses $P_d$, while the forward rates are the unknowns that build up $P_x$.

Theorem — Forecast Curve Bootstrap Step

Suppose $P_d$ is fully known, and $P_x(0,t_0),\ldots,P_x(0,t_{n-1})$ have already been determined from shorter-maturity instruments. Given the par rate $c_n^*$ of an $n$-period fixed-vs-$x$ swap with the same payment schedule as before, the par swap rate equation is: $$c_n^* \sum_{i=1}^n \delta_i^{fix} P_d(0,t_i) = \sum_{i=1}^n \delta_i^{flt} F_{x,i}\, P_d(0,t_i)$$ All terms for $i < n$ are known from previous bootstrap steps. The final forward rate $F_{x,n}$ is the only unknown and is solved directly: $$F_{x,n} = \frac{c_n^*\sum_{i=1}^n \delta_i^{fix}P_d(0,t_i) - \sum_{i=1}^{n-1}\delta_i^{flt}F_{x,i}\,P_d(0,t_i)}{\delta_n^{flt}P_d(0,t_n)}$$ The forecast curve is then extended one node further: $P_x(0,t_n) = P_x(0,t_{n-1}) / (1+\delta_n^{flt}F_{x,n})$.

Note that $P_x(0,t_n)$ has no independent meaning as a "discount factor" — it is purely the value that makes the forward-rate recursion produce the correct $F_{x,n}$. Different (arbitrary) choices of $P_x(0,0)$ would simply rescale the whole forecast curve without affecting any forward rate, which is the only quantity that matters.

Worked Multi-Curve Bootstrap Example

Consider a simplified two-period bootstrap (annual payments on both legs, $\delta_i^{fix}=\delta_i^{flt}=1$) for a 6M forecast curve, given an already-bootstrapped SOFR discount curve.

Example — Two-Step Forecast Curve Bootstrap

Suppose the SOFR discount curve gives $P_d(0,1)=0.97000$ and $P_d(0,2)=0.93500$, and the par rates for 1-year and 2-year fixed-vs-6M swaps are $c_1^*=3.00\%$ and $c_2^*=3.20\%$.

Year 1: with $P_x(0,0)=1$, the par rate equation for the 1-year swap is $c_1^* P_d(0,1) = F_{x,1}P_d(0,1)$, so trivially $F_{x,1}=c_1^*=3.00\%$, and: $$P_x(0,1) = \frac{1}{1+0.0300} = 0.97087$$

Year 2: using the theorem with $n=2$: $$F_{x,2} = \frac{c_2^*\big(P_d(0,1)+P_d(0,2)\big) - F_{x,1}P_d(0,1)}{P_d(0,2)} = \frac{0.0320\times(0.97000+0.93500) - 0.0300\times 0.97000}{0.93500}$$ $$F_{x,2} = \frac{0.0320\times 1.90500 - 0.02910}{0.93500} = \frac{0.06096 - 0.02910}{0.93500} \approx 3.408\%$$ The second-year forward rate on the 6M forecast curve is therefore approximately 3.41% — slightly higher than the 2-year par rate of 3.20%, reflecting the fact that the par rate is an average (annuity-weighted) of the two forward rates, and the first-year forward (3.00%) was below the par rate.
