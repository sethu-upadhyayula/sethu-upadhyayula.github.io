4.1 Pricing Bermudans with LMM

A Bermudan swaption gives its holder the right to enter an interest rate swap on any one of a discrete set of exercise dates. Pricing it requires modelling the joint evolution of all the forward rates that determine the underlying swap's value at every exercise date, and solving an optimal-stopping problem over that joint distribution. The LIBOR Market Model (LMM), introduced in Section 1.3, is the dominant framework for this task. This section explains why, and sketches how Monte Carlo simulation is combined with regression-based methods to handle the early-exercise feature.

Recap: The LIBOR Market Model

Remark — LMM Models Market-Observable Rates Directly

Recall from Section 1.3 that LMM specifies the joint dynamics of a discrete set of forward LIBOR (or, post-LIBOR, forward SOFR-compounded) rates $L_i(t) = L(t;T_i,T_{i+1})$, each lognormal (or displaced-lognormal/SABR-extended) under its own forward measure, with a full correlation matrix between the driving Brownian motions: $$dL_i(t) = \mu_i(t)\,dt + \sigma_i(t)\,L_i(t)\,dW_i^{(i)}(t), \qquad d\langle W^{(i)},W^{(j)}\rangle_t = \rho_{ij}\,dt$$ Unlike a 1- or 2-factor short-rate model, LMM's state is the entire vector of forward rates $(L_1(t),\ldots,L_n(t))$ — typically 10-40 rates for a 10-year structure with semi-annual or quarterly tenors. This high dimensionality is precisely what is needed to capture the joint behaviour of the swap rates underlying each exercise date of a Bermudan.

Why LMM for Bermudans?

Remark — Three Reasons LMM Dominates for Bermudan Swaptions

1. Direct calibration to the relevant vanillas. As discussed in Section 3.1, a Bermudan should be calibrated to the diagonal of co-terminal European swaptions spanning its exercise dates. Because each $L_i(t)$ in LMM is (to good approximation) lognormal with a volatility directly comparable to a caplet volatility, and swap rates are weighted combinations of the $L_i(t)$, LMM can be calibrated so that the model's implied volatilities for this entire diagonal of swaptions match the market simultaneously — something a 1- or 2-factor short-rate model cannot generally achieve exactly.

2. Realistic correlation structure. The correlation matrix $\rho_{ij}$ between forward rates is itself a calibratable input (fit to historical rate correlations or to the prices of correlation-sensitive instruments like CMS spread options). Since the value of the Bermudan's early-exercise feature depends on how the swap rate at one exercise date is related to the swap rate at the next, getting this correlation structure right — which a low-factor short-rate model cannot do, per the model-risk discussion in Section 3.2 — is central to pricing the Bermudan correctly.

3. A natural state space for Monte Carlo. The Bermudan payoff at each exercise date is a function of the then-prevailing forward rates (via the underlying swap's value). Since LMM simulates exactly these rates, the payoff at each exercise date is a simple, direct function of the simulated state — no further transformation from an abstract short-rate state variable is needed.

The Early-Exercise Problem

Pricing a Bermudan requires solving an optimal-stopping problem: at each exercise date $T_k$, the holder compares the immediate exercise value (the value of entering the underlying swap) against the value of continuing to hold the option. The Bermudan's price is the value of following the optimal exercise strategy.

Definition — Bermudan Value via Dynamic Programming

Let $h_k$ denote the intrinsic (exercise) value at exercise date $T_k$, and $V_k$ the value of the Bermudan at $T_k$ if not yet exercised. Working backwards from the final exercise date $T_N$ (where $V_N = h_N$), the value at each earlier date satisfies: $$V_k = \max\big(h_k,\; \mathbb{E}_k^{\mathbb{Q}}[\,D(T_k,T_{k+1})\,V_{k+1}\,]\big)$$ where $D(T_k,T_{k+1})$ is the stochastic discount factor and $\mathbb{E}_k^{\mathbb{Q}}[\cdot]$ denotes the conditional expectation given information at $T_k$. The price today is $V_0$ (suitably discounted). The term $\mathbb{E}_k^{\mathbb{Q}}[D(T_k,T_{k+1})V_{k+1}]$ — the continuation value — is the crux of the difficulty: it requires knowing $V_{k+1}$ as a function of the state at $T_k$, which is exactly what we are trying to compute.

Remark — Why This Is Hard for High-Dimensional Monte Carlo

A plain forward Monte Carlo simulation of LMM produces, at each exercise date, a large number of simulated paths and the corresponding forward rates along each path — but it gives no direct way to compute the continuation value $\mathbb{E}_k^{\mathbb{Q}}[D\,V_{k+1}]$ for a given path, because $V_{k+1}$ itself depends on the optimal future exercise decisions, which in turn depend on the (unknown) continuation values at $T_{k+1}$. PDE/lattice methods, which would naturally handle backward induction, are infeasible here because LMM's state space (10-40 correlated forward rates) is far too high-dimensional for a grid. This is precisely the situation regression-based Monte Carlo methods are designed for.

Regression-Based Methods (Longstaff-Schwartz)

Definition — The Least-Squares Monte Carlo (LSM) Approach

The Longstaff-Schwartz (LSM) algorithm estimates the continuation value at each exercise date as a function of the simulated state, using cross-sectional regression across simulated paths:

  1. Simulate $M$ paths of the LMM forward-rate vector $(L_1(t),\ldots,L_n(t))$ from today to the final exercise date $T_N$.
  2. At the final date $T_N$, set the realised cash flow on each path to the intrinsic value $h_N$ (exercise if positive, zero otherwise).
  3. Working backwards from $T_{N-1}$ to $T_1$: on each path, regress the (discounted) realised future cash flow from later exercise decisions against a set of basis functions of the current state (e.g. polynomials in the relevant swap rate, or in a small number of forward rates). The fitted regression value is an estimate of the continuation value $\mathbb{E}_k^{\mathbb{Q}}[D\,V_{k+1}]$ as a function of the state.
  4. On each path, compare the intrinsic value $h_k$ to the fitted continuation value (not the realised future cash flow itself, which would look ahead) to decide whether to exercise at $T_k$ on that path, and update the realised cash flow accordingly.
  5. After processing all exercise dates, average the resulting (discounted) cash flows across all $M$ paths to obtain the Bermudan price.
Remark — Why Regression Solves the Dimensionality Problem

The key insight is that the continuation value, while conceptually a function of the entire high-dimensional state $(L_1(t),\ldots,L_n(t))$, is in practice well-approximated by a function of a small number of relevant quantities — typically the underlying swap rate at that exercise date, perhaps together with one or two additional explanatory variables. Regressing on these low-dimensional basis functions, using the cross-section of simulated paths to estimate the regression coefficients, avoids ever needing to represent the continuation value on a high-dimensional grid. The exercise decision on each path then uses only this fitted low-dimensional approximation, and the law of large numbers ensures the final Monte Carlo average converges to the true price as $M\to\infty$ and the basis function set is enriched — making LSM-on-LMM the practical workhorse for Bermudan swaption pricing.
