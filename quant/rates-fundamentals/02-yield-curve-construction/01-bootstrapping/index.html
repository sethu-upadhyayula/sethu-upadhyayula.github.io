2.1 Bootstrapping the Yield Curve

The discount curve $T \mapsto P(0,T)$ is not directly observable. What is observable is a finite set of market quotes - deposit rates, futures or FRA rates, and swap rates - each of which constrains the discount curve in a specific way. Bootstrapping is the algorithm that inverts these quotes, instrument by instrument in order of maturity, to recover a discount curve that exactly reprices every input instrument.

Definition - Bootstrapping

Given a set of market instruments with maturities $T_1 < T_2 < \cdots < T_n$, bootstrapping constructs discount factors $P(0,T_1), \ldots, P(0,T_n)$ sequentially: at each step $i$, the discount factor $P(0,T_i)$ is chosen so that instrument $i$ is priced exactly at its market quote, using the discount factors $P(0,T_1), \ldots, P(0,T_{i-1})$ already determined (and interpolation for any cash flow dates that fall between pillars). The result is a discount curve that reprices the entire set of input instruments to zero (or to par, for swaps).

The Building Blocks: Deposits, FRAs/Futures, Swaps

A typical USD curve is built from three tiers of instruments, layered by maturity to balance liquidity against the desire for a smooth curve.

Remark - Instrument Tiers

Short end (cash deposits, 1 day - ~3 months). Money market deposit rates (or SOFR fixings) give the discount factor directly via simple compounding: $$P(0,T) = \frac{1}{1 + L(0,T)\,\delta(0,T)}$$ where $L(0,T)$ is the quoted deposit rate and $\delta$ is the Act/360 year fraction.

Middle (FRAs and futures, ~3 months - 2 years). Forward rate agreements and exchange-traded futures (e.g. SOFR or Eurodollar futures) quote forward rates over future periods $[T_{i-1}, T_i]$. Given $P(0,T_{i-1})$ from a previous step, the forward rate $F(0; T_{i-1}, T_i)$ implies: $$P(0,T_i) = \frac{P(0,T_{i-1})}{1 + F(0;T_{i-1},T_i)\,\delta_i}$$ Futures prices require a convexity adjustment (futures rates are not equal to forward rates) before being used in this step.

Long end (swap rates, 2 years and beyond). Par swap rates constrain the discount curve through the par rate equation of Section 1.1. Each new swap maturity introduces one new unknown discount factor, which is solved for given all shorter-maturity discount factors.

Sequential Solving for Discount Factors

The swap-rate step is the conceptual heart of bootstrapping: each additional par swap maturity adds exactly one new discount factor to solve for, using the previously bootstrapped discount factors for the earlier coupon dates.

Theorem - Bootstrapping a Swap Pillar

Let the par swap rate for maturity $T_n$ be $c_n$, with fixed payment dates $t_1 < \cdots < t_n = T_n$ and accrual fractions $\delta_i$. Suppose $P(0,t_1), \ldots, P(0,t_{n-1})$ have already been determined from shorter-maturity instruments. The par swap condition $$1 = c_n \sum_{i=1}^n \delta_i P(0,t_i) + P(0,t_n)$$ contains exactly one unknown, $P(0,t_n)$, which is recovered as: $$P(0,t_n) = \frac{1 - c_n \sum_{i=1}^{n-1} \delta_i P(0,t_i)}{1 + c_n \delta_n}$$

Applying this recursively from the shortest to the longest swap maturity yields the complete discount curve at all swap pillar dates.

Example - Two-Step Bootstrap

Suppose annual-pay par swap rates are $c_1 = 4.00\%$ and $c_2 = 4.50\%$, with $\delta_1 = \delta_2 = 1$.

Step 1 ($T_1 = 1$): $1 = c_1 \delta_1 P(0,1) + P(0,1) = (1+c_1)P(0,1)$, so $$P(0,1) = \frac{1}{1.04} = 0.96154$$

Step 2 ($T_2 = 2$): $1 = c_2[\delta_1 P(0,1) + \delta_2 P(0,2)] + P(0,2)$, so $$P(0,2) = \frac{1 - c_2 \cdot P(0,1)}{1 + c_2} = \frac{1 - 0.045 \times 0.96154}{1.045} = \frac{0.95673}{1.045} = 0.91553$$ Each step uses only previously solved discount factors - the defining feature of a bootstrap.

Handling Overlapping Instrument Maturities

In practice, the maturities of deposits, futures, and swaps overlap - a 2-year swap might mature on a date that also falls within the range of quoted futures contracts. Several practical issues arise from this overlap.

Remark - Overlap and Ordering Issues

Choosing which instruments to use. When multiple instruments imply discount factors over the same horizon, the curve builder must choose a single set of non-redundant, maximally liquid instruments - typically deposits at the very short end, futures or FRAs out to 1-3 years, and swaps beyond that, with a smooth "splice" between tiers.

Stub periods. A futures contract's reference period may not align exactly with the previous pillar date, requiring interpolation of $P(0,T_{i-1})$ at a non-pillar date - which feeds back into the choice of interpolation scheme (Section 2.2).

Jacobian and re-bootstrapping. If an intermediate quote changes, all downstream discount factors that were computed using interpolated values at that pillar must be recomputed. Production curve-building systems solve the entire system of equations jointly (often via Newton's method on the full pillar vector) rather than purely sequentially, precisely to handle these interdependencies robustly.

Multi-Curve Framework: OIS Discounting vs. Forward Curves

Since the 2008 financial crisis, a single discount curve no longer suffices. Collateralized derivatives are discounted using the overnight rate that accrues on posted collateral (OIS/SOFR), while floating coupons are projected using a separate forward or projection curve appropriate to the index (e.g. 3-month Term SOFR, or historically 3-month LIBOR).

Definition - Multi-Curve Framework

In the multi-curve framework, valuation uses (at least) two curves:

  • Discount curve $P_{OIS}(0,T)$ - built by bootstrapping OIS/SOFR swap rates, used to discount all cash flows of a collateralized trade.
  • Forward (projection) curve $P_{fwd}(0,T)$ - built by bootstrapping instruments referencing the floating index (e.g. SOFR-based term instruments or, historically, LIBOR), used only to compute the forward rates that determine floating coupon amounts: $$F(0;T_{i-1},T_i) = \frac{1}{\delta_i}\left(\frac{P_{fwd}(0,T_{i-1})}{P_{fwd}(0,T_i)} - 1\right)$$

A floating leg cash flow is computed using the forward curve but discounted using the OIS curve: $$PV_{\text{float leg}} = \sum_i \delta_i\, F(0;T_{i-1},T_i)\, P_{OIS}(0,t_i)$$ Each curve is bootstrapped separately, but the projection curve bootstrap typically requires the OIS discount curve as an input (since instrument valuations on that curve are also discounted at OIS), creating a layered dependency between curves.

Remark - Why Multi-Curve Matters

Before the financial crisis, LIBOR and OIS rates tracked each other closely (a few basis points apart), so a single LIBOR-based curve was used for both discounting and projection. After 2008, LIBOR-OIS spreads widened to over 300bp at times, reflecting bank credit and liquidity risk embedded in LIBOR but absent from collateralized OIS rates. Using a single curve for both purposes would misprice swaps by failing to reflect the true cost of funding (collateral) versus the floating index being referenced. The multi-curve framework - and its continuation in the SOFR era, discussed further in the Interest Rate Swaps notes - corrects this by cleanly separating the discounting and projection roles.
