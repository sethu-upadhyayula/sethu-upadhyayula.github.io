1. Software Development Methodologies

Before a single line of code is written, a team must decide how it will actually organize the work of building software — this chapter covers the major methodologies that have shaped how software projects are run.

The Waterfall Model

Definition — Waterfall

The waterfall model organizes development into a strict sequence of phases — requirements, design, implementation, testing, deployment — each completed fully before the next begins, with limited expectation of revisiting an earlier phase once passed. It works reasonably well when requirements are genuinely stable and well understood upfront, but struggles when requirements change during development, since the model provides no natural mechanism for incorporating that change gracefully.

Agile Methodologies

Definition — Agile

Agile methodologies replace waterfall's single long sequence with short, repeated cycles (iterations or sprints), each delivering a working, incrementally improved piece of software and incorporating feedback before the next cycle begins — explicitly designed to accommodate changing requirements throughout a project rather than assuming they're fully known in advance.

Scrum

Scrum is the most widely adopted specific Agile framework, organizing work into fixed-length sprints (commonly one to four weeks), with a prioritized backlog of work items, a daily short synchronization meeting (standup), and a review and retrospective at the end of each sprint — providing concrete structure and ceremony around Agile's general principles.

Kanban

Kanban takes a different approach to the same Agile spirit — rather than fixed-length sprints, work items flow continuously through a visualized pipeline of stages (e.g., To Do, In Progress, Done), with explicit limits on how much work can be in progress at each stage simultaneously, aiming to expose and address bottlenecks directly rather than batching work into sprint-sized chunks.

DevOps and Continuous Delivery

Remark — Extending Agile Beyond Development

DevOps extends Agile's iterative philosophy beyond development itself, into how software is deployed and operated — emphasizing close collaboration (or even merged responsibility) between development and operations teams, automated deployment pipelines (Chapter 7), and rapid, frequent, low-risk releases rather than large, infrequent, high-risk ones, a theme this subject returns to directly in Chapter 7.
