5. Intermediate Representations

Once semantic analysis (Chapter 4) confirms a program is valid, most compilers translate its abstract syntax tree into an intermediate representation (IR) — a form more convenient for the optimization (Chapter 6) and code generation (Chapter 7) phases to work with than the original AST.

Why an Intermediate Representation?

Remark — The Purpose of an IR

An IR is deliberately designed to be simpler and more uniform than a source language's full syntax (Chapter 3) — collapsing many different high-level constructs (for-loops, while-loops, and other looping forms, for instance) into a small set of common lower-level primitives — so that optimization passes (Chapter 6) can be written once against this simpler, uniform representation, rather than needing to separately handle every syntactic variant a source language allows.

Three-Address Code

Definition — Three-Address Code

Three-address code is a common IR form in which each instruction performs at most one operation, with at most three operands (typically two sources and one destination) — for instance, t1 = b * c followed by t2 = a + t1, rather than the single compound expression a + b * c — breaking complex expressions down into a sequence of simple, uniform steps that are straightforward to analyze and transform.

Static Single Assignment Form

Definition — SSA Form

Static single assignment (SSA) form requires that every variable be assigned exactly once in the program text — if a variable would naturally be reassigned multiple times, each assignment instead creates a distinct new versioned variable — making data-flow relationships explicit and direct in the IR itself (a use of a variable can only ever refer to its one, unique defining assignment), which considerably simplifies many optimization algorithms (Chapter 6) compared to reasoning about a variable that might have been assigned at any of several different points.

Control Flow Graphs

A control flow graph (CFG) represents a function's possible execution paths as a graph of basic blocks (maximal straight-line instruction sequences with no internal branching) connected by edges representing possible jumps between them — the standard structure both optimization passes (Chapter 6) and code generation (Chapter 7) operate over, capturing exactly the control-flow information needed for analyses like determining which optimizations are safe to apply where.

Multiple Levels of IR

Many production compilers use several IRs at different levels of abstraction in sequence — a higher-level IR closer to the source language for early, more semantically-aware optimizations, progressively lowered to representations closer to actual machine instructions as compilation proceeds toward the code generation phase covered in Chapter 7.
