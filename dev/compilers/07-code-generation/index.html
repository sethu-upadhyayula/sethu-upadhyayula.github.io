7. Code Generation

After optimization (Chapter 6), the compiler's back end translates the intermediate representation into actual instructions for a specific target processor (Computer Architecture) — this chapter covers the major tasks involved in that final translation.

Instruction Selection

Definition — Instruction Selection

Instruction selection chooses which target-architecture instructions (Chapter 1 of Computer Architecture) implement each IR operation — a task considerably more involved on CISC architectures with many possible, differently-priced ways to express the same computation (Chapter 1 of Computer Architecture), often approached by matching IR patterns against a library of known instruction sequences and selecting the lowest-cost matching option.

Register Allocation

Definition — Register Allocation

Register allocation assigns the (typically far more numerous) temporary values used in the IR to the processor's small, fixed number of physical registers (Chapter 1 of Computer Architecture) — a classic approach models this as a graph coloring problem, where temporaries needed simultaneously ("interfering" with each other) must be assigned different registers, with spilling (temporarily storing a value in main memory instead) required whenever there aren't enough physical registers to go around.

Instruction Scheduling

Instruction scheduling reorders generated instructions (while preserving the program's correctness and data dependencies) to better exploit the target processor's pipeline (Chapter 3 of Computer Architecture) — for instance, separating a load instruction from the instruction that immediately uses its result, giving the memory access more time to complete before that result is actually needed, reducing pipeline stalls.

Calling Conventions

Definition — Calling Convention

A calling convention specifies precisely how function calls are implemented at the machine level — which registers hold arguments and return values, how the stack is used for a function's local variables and any spilled register values (above), and whose responsibility it is to preserve which registers across a call — a convention that must be followed consistently by every piece of code (from every source language and even every compiler) that needs to correctly call into, or be called from, other compiled code.

Assembly and Object Code

Code generation typically produces either human-readable assembly (further processed by a separate assembler into machine code) or object code directly — machine instructions along with the symbol and relocation information a linker needs to combine multiple separately compiled files (and any required libraries) into a single final executable, resolving cross-file function and variable references in the process.
