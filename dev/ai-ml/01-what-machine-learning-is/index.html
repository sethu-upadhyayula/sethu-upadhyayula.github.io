1. What Machine Learning Is

Rather than a programmer explicitly writing out the rules a program should follow, machine learning builds systems that infer those rules (or an approximation of them) automatically from data — this chapter surveys the field's basic shape before later chapters look at specific techniques.

Supervised, Unsupervised, and Reinforcement Learning

The Three Broad Categories
  • Supervised learning — learning a mapping from inputs to known, labeled outputs (Chapter 2), such as predicting a house price from its features.
  • Unsupervised learning — finding structure in data with no labels at all (Chapter 3), such as grouping similar customers together.
  • Reinforcement learning — learning to make sequential decisions from feedback (rewards) rather than labeled examples (Chapter 7), such as an agent learning to play a game.

Training Data and Generalization

A model is fit to a training set of example data, but the actual goal is generalization — performing well on new, previously unseen data, not just memorizing the training examples themselves. A model that fits its training data extremely well but fails to generalize is said to be overfit, a central concern revisited in Chapter 4.

Features

The individual measurable inputs a model is given — a house's square footage, an email's word frequencies — are called features. Historically, choosing and engineering good features by hand was a large part of applied machine learning; a major appeal of the deep learning approach covered in Chapter 5 is that it can learn useful features directly from raw data, reducing the need for this manual step.

Why This Is Only a Survey

Remark — Scope of This Subject

Machine learning rests heavily on linear algebra, probability and statistics, and optimization (all covered in Math and Quant) — a proper treatment of even a single technique in this subject, such as backpropagation (Chapter 5) or gradient descent (Chapter 4), can occupy an entire dedicated course. This subject aims only to convey what each idea is and roughly how it works, not to derive or implement it.
