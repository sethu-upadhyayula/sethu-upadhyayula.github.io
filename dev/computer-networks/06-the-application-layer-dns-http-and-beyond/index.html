6. The Application Layer: DNS, HTTP, and Beyond

At the top of the network stack (Chapter 1), application-layer protocols define the actual message formats and conventions specific services use - this chapter covers two of the most foundational: DNS, which lets us use names instead of raw IP addresses, and HTTP, the protocol underlying the web.

DNS: The Domain Name System

Definition - DNS

The Domain Name System (DNS) translates human-readable domain names (like a website's address) into the numeric IP addresses (Chapter 3) actually needed to route packets to it - implemented as a distributed, hierarchical database, queried via UDP (Chapter 4) for speed, with results cached at multiple levels (in the browser, operating system, and intermediate resolvers) to avoid repeating the same lookup unnecessarily.

DNS Resolution Hierarchy

Resolving a domain name typically involves querying a hierarchy of servers - root servers, then top-level-domain servers, then the specific domain's authoritative name servers - though in practice, a recursive resolver (often provided by an ISP or a public DNS service) handles this multi-step process on the client's behalf, returning only the final answer.

HTTP

Definition - HTTP

HTTP (Hypertext Transfer Protocol) is a request-response protocol built on top of TCP (Chapter 4), in which a client sends a request (specifying a method such as GET or POST, a target resource, and headers) and a server returns a response (a status code, headers, and typically a body) - the foundational protocol of the World Wide Web, and the basis for the vast majority of modern web APIs as well.

HTTP Evolution: Keep-Alive, Multiplexing, and QUIC

Early HTTP opened a new TCP connection per request; modern HTTP reuses connections across multiple requests (keep-alive) and, in HTTP/2, multiplexes many requests over a single connection simultaneously to avoid head-of-line blocking at the application level. HTTP/3 goes further, running over QUIC (built on UDP rather than TCP) specifically to avoid a different form of head-of-line blocking that occurs at the TCP layer itself when packets are lost.

Other Application-Layer Protocols

Further Application-Layer Examples
  • SMTP - email transmission between mail servers.
  • SSH - secure remote shell access and file transfer.
  • WebSocket - a persistent, full-duplex connection over an initial HTTP handshake, used for real-time bidirectional communication that a simple request-response model doesn't suit well.
