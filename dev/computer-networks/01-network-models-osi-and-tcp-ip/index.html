1. Network Models: OSI and TCP/IP

Networking is organized into layers, each responsible for a distinct part of getting data from one machine to another - this chapter introduces the two standard layered models used to reason about that organization, before the rest of this subject works through each layer in turn.

Why Layering?

Remark - The Value of Layering

Layering lets each part of the network stack be designed, implemented, and reasoned about somewhat independently - a web application (application layer) does not need to know whether the underlying physical link is Ethernet or Wi-Fi, and a new physical medium can be introduced without requiring changes to how applications communicate - as long as each layer presents a stable interface to the layer above it.

The OSI Model

Definition - The Seven OSI Layers

The OSI (Open Systems Interconnection) model divides networking into seven layers: physical, data link, network, transport, session, presentation, and application. It is primarily used today as a conceptual teaching and reference framework rather than a literal description of how most real systems are implemented.

The TCP/IP Model

Definition - The Four TCP/IP Layers

The TCP/IP model, which more closely reflects how the actual internet is built and is the model this subject follows, condenses networking into four layers: link (Chapter 2), internet (Chapter 3), transport (Chapter 4), and application (Chapter 6) - collapsing OSI's separate session and presentation layers into the application layer, and its physical and data link layers into a single link layer.

Encapsulation

As data moves down the stack from application to physical layer, each layer wraps ("encapsulates") the data from the layer above with its own header (and sometimes trailer) information - an application's data becomes a transport-layer segment, then a network-layer packet, then a link-layer frame - with the reverse process (decapsulation) stripping each header away as the data moves back up the stack at the receiving end.

Protocols as Layer Implementations

A protocol is a concrete, precisely specified implementation of one layer's responsibilities - IP (Chapter 3) implements the internet layer, TCP and UDP (Chapter 4) implement the transport layer, and HTTP and DNS (Chapter 6) implement specific application-layer services - each covered in the corresponding chapter of this subject.
