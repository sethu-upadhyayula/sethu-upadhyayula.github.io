7. Network Security Fundamentals

Every layer of the network stack covered so far in this subject transmits data that, without additional protection, can potentially be intercepted, read, or tampered with in transit - this chapter covers the core mechanisms used to secure network communication.

TLS: Encrypting Traffic in Transit

Definition - TLS

TLS (Transport Layer Security) sits between the transport layer (Chapter 4) and application layer (Chapter 6), encrypting data in transit and authenticating the server (and optionally the client) via digital certificates - the mechanism behind HTTPS, which is simply HTTP (Chapter 6) run over a TLS-encrypted connection rather than a plain TCP one.

The TLS Handshake

A TLS connection begins with a handshake in which client and server agree on a shared encryption algorithm and securely establish a shared symmetric key (used for the fast bulk encryption of the actual data, since asymmetric encryption is considerably more computationally expensive), with the server's certificate - verified against a trusted certificate authority - providing assurance that the client is actually talking to the server it intends to, not an impersonator.

Firewalls

Definition - Firewall

A firewall inspects network traffic and permits or blocks it according to a configured rule set - a simple packet-filtering firewall decides based on packet headers alone (source/destination IP and port, Chapters 3–4), while a stateful firewall tracks ongoing connections and can make more nuanced decisions based on a connection's established context rather than evaluating every packet in isolation.

Common Network Attacks

Attack Types Network Security Defends Against
  • Denial of service (DoS/DDoS) - overwhelming a target with traffic or requests to exhaust its resources and deny service to legitimate users.
  • Man-in-the-middle attacks - an attacker secretly intercepts (and potentially alters) communication between two parties who believe they are communicating directly, which TLS's authentication step is specifically designed to prevent.
  • Spoofing - forging a packet's source address or a DNS response (Chapter 6) to impersonate a trusted party.

VPNs

A VPN (Virtual Private Network) creates an encrypted tunnel between a client and a remote network, making traffic appear (and, with respect to confidentiality, behave) as though the client were directly connected to that remote private network - commonly used both for secure remote access to an organization's internal network and, more generally, to encrypt and reroute traffic across an otherwise untrusted network path.
