3. The Network Layer: IP and Routing

Where the data link layer (Chapter 2) handles communication within a single local network, the network layer handles the much larger problem of getting a packet across many interconnected networks to a distant destination anywhere on the internet.

IP Addressing

Definition - IP Address

An IP address uniquely identifies a device's network interface for routing purposes. IPv4 addresses are 32 bits (commonly written as four decimal numbers, e.g. 192.168.1.1), providing roughly 4.3 billion possible addresses - a number long since exhausted for direct global allocation, driving both the widespread use of network address translation (NAT, allowing many devices to share one public IP address) and the ongoing transition to IPv6, whose 128-bit address space is vastly larger.

Subnetting

Subnetting divides an IP address into a network portion and a host portion (using a subnet mask), allowing addresses to be allocated and routed hierarchically - a router only needs to know how to reach the correct network as a whole, not every individual host address within it, a structure essential to keeping internet-scale routing tables tractable.

Routing

Definition - Routing

Routing is the process of determining the path a packet takes from source to destination across a series of intermediate routers, each forwarding the packet one hop closer based on its own routing table - a mapping from destination network to the next-hop router to forward toward, built and updated by routing protocols exchanging reachability information with neighbouring routers.

Routing Protocols

Within a single organization's network, interior gateway protocols (such as OSPF) compute efficient routes using link-state or distance-vector algorithms. Between separate organizations' networks across the wider internet, BGP (Border Gateway Protocol) exchanges reachability information between autonomous systems, incorporating policy considerations (which routes an organization is willing to use or advertise) alongside pure path-length optimization.

Fragmentation and Packet Size

Remark - Maximum Transmission Unit

Different physical links (Chapter 2) support different maximum frame sizes - the maximum transmission unit (MTU). If a packet exceeds the MTU of a link it must cross, it may need to be fragmented into smaller pieces (reassembled at the destination) - a costly process that modern practice increasingly avoids through path MTU discovery, proactively finding the smallest MTU along a path and sizing packets accordingly from the start.
