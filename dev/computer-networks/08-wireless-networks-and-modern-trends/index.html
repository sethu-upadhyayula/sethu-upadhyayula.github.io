8. Wireless Networks and Modern Trends

This closing chapter covers what changes when the physical layer (Chapter 2) is a shared radio channel rather than a dedicated wire, and surveys a few current directions in how networks are built and operated.

Wi-Fi and Shared Wireless Media

Definition - Contention-Based Wireless Access

Unlike switched wired Ethernet (Chapter 2), Wi-Fi devices share a genuinely broadcast radio medium - multiple devices transmitting simultaneously on the same channel can interfere with and corrupt each other's signals (collision), requiring a medium access control protocol (such as CSMA/CA, which attempts to detect the channel is busy and avoid transmitting during that time) to coordinate access, a genuinely harder coordination problem than switched wired networks generally face.

Cellular Networks

Cellular (mobile) networks divide geography into cells, each served by a base station, with dedicated protocols managing a device's handoff between cells as it moves and allocating limited radio spectrum efficiently among many simultaneous users - successive cellular generations (4G, 5G) have progressively increased achievable bandwidth and reduced latency, increasingly blurring the line between cellular and traditional wired broadband performance for many use cases.

Content Delivery Networks

Definition - CDN

A content delivery network (CDN) caches content at servers geographically distributed close to end users, reducing the network distance (and hence latency) data must travel compared to always fetching it from a single, potentially distant origin server - a widely used technique for improving web performance at global scale, and one that interacts directly with DNS (Chapter 6), which is typically used to direct a given user to their geographically nearest CDN server.

Software-Defined Networking

Software-defined networking (SDN) separates a network's control logic (deciding how traffic should be routed) from the underlying forwarding hardware (which simply executes those decisions), centralizing routing and policy decisions in software rather than distributing them entirely across many independently configured routers (Chapter 3) - offering more flexible, programmable network management, particularly valuable in large data center networks.

Networking as the Foundation for Distributed Systems

Remark - Where This Leads Next

Every mechanism covered across this subject - addressing (Chapter 3), reliable transport (Chapters 4–5), and application protocols (Chapter 6) - provides the basic communication substrate that Distributed Systems builds on, where the focus shifts from "how do two machines communicate" to "how do many machines coordinate to behave as a single, consistent system despite that communication being imperfect and sometimes slow."
