1. The Relational Model

The relational model, introduced by Edgar Codd in 1970, represents data as simple tables - a deceptively simple idea that has remained the dominant way to organize structured data for over half a century.

Relations, Tuples, and Attributes

Definition - Relation

A relation (informally, a table) is a set of tuples (rows), each assigning a value to every attribute (column) defined by the relation's schema. Because a relation is mathematically a set, in strict relational theory it cannot contain duplicate tuples and has no inherent row ordering - properties that most real SQL databases relax somewhat in practice for practical convenience.

Keys

Types of Keys
  • Candidate key - a minimal set of attributes that uniquely identifies each tuple in a relation.
  • Primary key - the candidate key chosen to be the relation's main identifier, used by other relations to reference it.
  • Foreign key - an attribute (or set of attributes) in one relation that references the primary key of another, establishing a relationship between the two and enforced by referential integrity constraints.

Relational Algebra

Definition - Relational Algebra

Relational algebra is a formal set of operations - selection (filtering rows), projection (selecting columns), union, difference, and join (combining relations based on a matching condition) chief among them - that take one or more relations as input and produce a new relation as output. Every SQL query (Chapter 2) can, at least in principle, be translated into an equivalent relational algebra expression, and query optimizers (Chapter 7) work by transforming and reordering exactly these kinds of operations.

Schemas and Instances

A relation's schema - its name and the names and types of its attributes - is distinct from its instance, the actual set of tuples currently stored; a schema is relatively stable over a database's lifetime, while its instance changes constantly as data is inserted, updated, and deleted.

Why the Relational Model Endured

Remark - A Durable Abstraction

The relational model's enduring success owes much to its strong mathematical foundation (making formal reasoning about queries and correctness possible) combined with a genuinely simple, intuitive data structure (a table) - a combination that later data models, including many NoSQL systems (Chapter 8), have often found themselves re-approximating in one form or another rather than escaping entirely.
