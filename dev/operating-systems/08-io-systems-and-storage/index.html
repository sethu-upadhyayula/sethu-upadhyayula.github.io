8. I/O Systems and Storage

This closing chapter covers how the operating system communicates with the physical devices - disks, network cards, and other peripherals - that everything in this subject ultimately depends on, and the storage hardware those devices are built around.

Device Drivers

Definition - Device Driver

A device driver is OS-level software that translates the OS's generic I/O requests into the specific commands a particular hardware device understands, presenting a uniform interface (e.g., "read this block" or "send this network packet") to the rest of the OS regardless of which specific manufacturer's hardware is actually installed.

Polling versus Interrupts

A CPU can learn that a device needs attention either by polling (repeatedly checking the device's status register) or via interrupts (the device signals the CPU directly when it needs attention, allowing the CPU to do other useful work in the meantime) - interrupts are generally far more efficient for devices with unpredictable or infrequent events, while polling can be more efficient for devices generating a continuous, predictable stream of requests.

Direct Memory Access

Definition - DMA

Direct memory access (DMA) allows a device to transfer data directly to or from main memory without the CPU needing to copy every byte itself - the CPU simply initiates the transfer and is interrupted once it completes, freeing it to perform other work during what would otherwise be a slow, CPU-mediated data transfer.

Hard Disk Drives and Scheduling

Traditional hard disk drives (HDDs) store data on rotating magnetic platters, with a physically moving read/write head - meaning the order in which disk blocks are read matters considerably for performance, motivating disk scheduling algorithms (such as SCAN, which sweeps the head across the disk in one direction, servicing requests along the way, rather than jumping unpredictably between distant requested blocks) that minimize costly physical head movement (seek time).

Solid-State Drives

Remark - Why SSDs Change the Calculus

Solid-state drives (SSDs) store data electronically rather than on rotating magnetic media, with no physical seek time at all and much faster random access - significantly reducing the performance benefit of the disk-scheduling algorithms designed around HDDs' mechanical constraints, though SSDs introduce their own considerations (limited write endurance per cell, and the need for wear leveling to spread writes evenly across the device) that HDDs never had to contend with.

Buffering and Caching

The OS maintains an in-memory buffer cache of recently accessed disk blocks, serving repeated reads directly from memory rather than the much slower underlying storage device, and batching or delaying writes where safely possible - a caching layer that sits between the file system abstractions of Chapter 7 and the physical storage hardware covered in this chapter, and one of the most impactful practical optimizations in the entire I/O stack.
