6. Virtual Memory

Building on the paging mechanism from Chapter 5, virtual memory lets a system run processes whose combined memory demands exceed the machine's actual physical RAM, by keeping only currently needed pages resident in memory at any moment.

Demand Paging

Definition - Demand Paging

Demand paging loads a page into physical memory only when it is actually accessed, rather than loading a process's entire address space upfront. Accessing a page not currently in physical memory triggers a page fault - the OS pauses the faulting process, locates the required page (typically on disk), loads it into a free physical frame, updates the page table (Chapter 5), and resumes the process.

Page Replacement

When physical memory is full and a page fault occurs, the OS must choose an existing resident page to evict to make room - the choice of page replacement algorithm significantly affects overall system performance, since evicting a page that will be needed again very soon forces an expensive, avoidable subsequent page fault.

Common Page Replacement Algorithms
  • FIFO - evicts the longest-resident page; simple, but can perform poorly and even suffer Bélády's anomaly (more physical memory occasionally causing more page faults).
  • Least Recently Used (LRU) - evicts the page not accessed for the longest time, generally performing well but requiring hardware or software support to actually track access recency efficiently.
  • Clock (second-chance) - an efficient practical approximation of LRU, using a single "recently used" bit per page and a circular scan rather than tracking a full access history.

Thrashing

Remark - Thrashing

If too many processes are active relative to available physical memory, a system can enter thrashing - spending most of its time servicing page faults and evicting pages that are needed again almost immediately, with very little actual useful work getting done - a pathological state that reducing the number of concurrently active processes (rather than any cleverer page replacement algorithm alone) is typically needed to escape.

The Working Set Model

The working set of a process is the set of pages it has referenced within a recent time window - a practical model for estimating how much physical memory a process actually needs to run efficiently at a given moment, and one basis for deciding how many processes a system can support concurrently before risking thrashing.

Memory-Mapped Files

Virtual memory's page-fault machinery is also used to implement memory-mapped files - mapping a file directly into a process's address space so that ordinary memory reads and writes transparently become file reads and writes, handled via the same demand-paging mechanism described above, connecting virtual memory directly to file system access (Chapter 7).
