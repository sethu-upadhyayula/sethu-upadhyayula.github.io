7. Parallelism Beyond a Single Core: Multicore and SIMD

As instruction-level parallelism within a single core reached practical and power-related limits (Chapter 6), processor designers turned to two complementary strategies for continued performance growth — multiple cores, and instructions that operate on multiple data elements at once.

Why Single-Core Speed Growth Slowed

Remark — The End of Simple Clock-Speed Scaling

For decades, processor performance improved substantially just from increasing clock speed and exploiting ever-deeper pipelining and ILP (Chapters 3 and 6) — a trend that slowed markedly once power consumption and heat dissipation (power scales worse than linearly with clock frequency) became binding practical constraints, redirecting the industry's performance gains toward adding more cores rather than making any single core faster.

Multicore Processors

Definition — Multicore Processor

A multicore processor integrates multiple independent processing cores on a single chip, each capable of executing its own instruction stream, coordinated through the cache coherence mechanisms covered in Chapter 5 when accessing shared memory. Realizing multicore's performance potential requires software explicitly written to exploit multiple threads (Chapter 1 of Operating Systems) — a single-threaded program gains essentially nothing from additional cores beyond the first.

SIMD: Data-Level Parallelism

Definition — SIMD

SIMD (Single Instruction, Multiple Data) instructions apply the same operation simultaneously to multiple data elements packed into a single wide register — for example, adding four pairs of numbers in one instruction rather than four separate instructions — well suited to data-parallel workloads such as image and signal processing, and a major contributor to the raw computational throughput of modern GPUs.

GPUs: Massive Data Parallelism

GPUs (Graphics Processing Units) take the SIMD idea considerably further, with thousands of relatively simple cores executing the same instruction stream across massive amounts of data simultaneously — originally built for graphics rendering (which is naturally embarrassingly data-parallel across pixels), but now widely repurposed for general-purpose data-parallel computation, most notably the large-scale matrix operations underlying modern AI/ML (AI/ML) workloads.

Amdahl's Law

Amdahl's Law

Amdahl's law quantifies a hard limit on parallel speedup: if a fraction $p$ of a program's execution time can be parallelized (and the remaining $1-p$ is inherently sequential), the maximum possible speedup from infinite parallel processing power is $\frac{1}{1-p}$ — meaning even a small sequential portion of a program can severely cap the benefit of adding ever more cores, a sobering practical constraint on how much multicore parallelism (or any other kind) can actually help any specific program.
