6. Instruction-Level Parallelism and Branch Prediction

Beyond the basic overlap pipelining (Chapter 3) provides, modern processors extract considerably more parallelism from a single sequential instruction stream — this chapter covers how, along with the branch prediction techniques that make it practical.

Superscalar Execution

Definition — Superscalar Processor

A superscalar processor can fetch, decode, and execute more than one instruction per clock cycle, using multiple parallel execution units (e.g., separate units for integer arithmetic, floating point, and memory access) — extracting instruction-level parallelism (ILP) from a program by identifying independent instructions that can genuinely execute simultaneously, constrained by the data hazards covered in Chapter 3.

Out-of-Order Execution

Definition — Out-of-Order Execution

Out-of-order execution allows the processor to execute instructions in an order different from the program's original sequence, whenever doing so avoids stalling on a not-yet-available operand — instructions are dispatched to execution units as their operands become ready, with a reorder buffer ensuring results are still committed (written back) in the original program order, preserving correct program behaviour despite the internal reordering.

Branch Prediction

As introduced in Chapter 3, a branch predictor guesses whether a conditional branch will be taken (and to which target) before it is actually resolved, allowing the pipeline to speculatively continue fetching and even executing instructions down the predicted path. Modern predictors use dynamic history-based schemes — tracking a given branch's recent outcomes, and sometimes correlating with the outcomes of other nearby branches — achieving prediction accuracy well above 90% for typical program branches.

Speculative Execution

Remark — Speculation and Security

Speculative execution — executing instructions before knowing for certain they should run, based on a branch prediction — is essential to modern processor performance, but has also been the root cause of significant security vulnerabilities (such as Spectre and Meltdown), where an incorrectly speculated path can leave observable side effects (in cache state, Chapter 4) even after being architecturally "rolled back," potentially leaking sensitive data across security boundaries an attacker was never meant to be able to observe.

The Limits of ILP

Instruction-level parallelism has practical limits — a program's true data dependencies eventually bound how many instructions can genuinely execute in parallel regardless of hardware sophistication, a limitation that has driven processor designers toward the multicore and vector-parallelism approaches covered in Chapter 7 to continue improving performance once ILP extraction alone reached diminishing returns.
