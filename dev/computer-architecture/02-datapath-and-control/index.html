2. Datapath and Control: How a CPU Executes an Instruction

Given an ISA's defined instruction vocabulary (Chapter 1), this chapter covers the actual hardware — the datapath and control logic — that carries out each instruction step by step.

The Fetch-Decode-Execute Cycle

Definition — The Basic Instruction Cycle

A processor executes each instruction through a repeating cycle: fetch (retrieve the next instruction from memory, using the program counter to know where), decode (determine which operation the instruction specifies and which registers/values it needs), execute (perform the operation, typically in the arithmetic logic unit), and write-back (store the result to a register or memory).

The Datapath

The datapath is the hardware — registers, the arithmetic logic unit (ALU), and the buses connecting them — through which actual data values flow as an instruction is executed; different instructions route data through the datapath differently (an arithmetic instruction routes register values through the ALU, while a load instruction routes a computed address to memory and the result back to a register).

Control Logic

Definition — Control Unit

The control unit generates the specific signals that configure the datapath correctly for whichever instruction is currently being decoded — determining which registers to read, whether the ALU should add or subtract, and whether the result should be written to a register or memory — effectively translating an instruction's encoded bits (Chapter 1) into the specific hardware configuration needed to carry it out.

Single-Cycle versus Multi-Cycle Implementations

A single-cycle processor implementation completes every instruction in exactly one clock cycle, simple to design and reason about but forced to run at the speed of its slowest possible instruction. A multi-cycle implementation instead breaks execution into several shorter cycles, allowing simple instructions to finish faster while more complex ones take proportionally more cycles — a precursor to the pipelined approach covered in Chapter 3, which overlaps these stages across multiple instructions simultaneously rather than running them fully sequentially for one instruction at a time.

Hardwired versus Microprogrammed Control

Remark — Two Ways to Implement Control Logic

Control logic can be hardwired (implemented directly as fixed digital logic circuits, fast but inflexible) or microprogrammed (implemented as a small internal program, a "microcode," that itself gets fetched and executed to generate control signals) — CISC processors (Chapter 1) with complex instructions have historically favoured microprogramming for its flexibility, while simpler RISC instructions are often efficient enough to hardwire directly.
